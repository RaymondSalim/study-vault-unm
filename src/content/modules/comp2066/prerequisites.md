# Prerequisites

## Required Background Knowledge

| Area | Topics Needed | Why |
|------|---------------|-----|
| Programming | Loops, recursion, arrays, linked lists, basic OOP | Implementing and understanding algorithm pseudocode |
| Mathematics | Logarithms, exponents, summation notation, basic proofs | Complexity analysis, recurrence relations, induction proofs |
| Basic Data Structures | Arrays, linked lists, stacks, queues | Foundation for trees, heaps, graphs, hash tables |
| Introductory Algorithms | Linear search, bubble sort, basic recursion | Starting point; this module builds more efficient alternatives |

## Helpful Prior Modules

- **COMP1013 - Algorithms & Data Structures (Year 1)** -- Basic data structures, simple sorting, recursion
- **COMP1015 - Mathematical Foundations** -- Proof by induction, logarithms, set theory
- **COMP1012 - Programming** -- Recursion, pointers/references, implementing data structures
- **Any Maths module** -- Comfort with mathematical notation and formal reasoning

## Key Concepts You Should Already Know

1. How recursion works (base case, recursive case, call stack)
2. What a linked list is and how to traverse it
3. Logarithms: log_2(1024) = 10; log_2(n) is the number of times you can halve n
4. What a stack and queue are (LIFO vs FIFO)
5. Summation: 1 + 2 + ... + n = n(n+1)/2

## Self-Assessment Test

Answer these before starting the module. If you struggle with more than 2, review the prerequisites first.

| # | Question | Expected Answer |
|---|----------|-----------------|
| 1 | What is log_2(32)? | 5 (because 2^5 = 32) |
| 2 | Write a recursive function to compute factorial(n). | `factorial(0) = 1; factorial(n) = n * factorial(n-1)` |
| 3 | What is the sum 1 + 2 + 3 + ... + 100? | 100 x 101 / 2 = 5050 |
| 4 | If an algorithm does n/2 operations in a loop, what is its Big-O? | O(n) -- constant factors are dropped |
| 5 | What data structure uses FIFO ordering? | A queue (first in, first out) |
