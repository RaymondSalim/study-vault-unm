## Common Exam Traps

### 1. Ethical Theory Confusion

| Trap | Correct |
|------|---------|
| "Consequentialism and utilitarianism are different theories" | Utilitarianism is the major branch OF consequentialism |
| "Deontology cares about consequences" | Deontology judges actions by rules/duties, NOT outcomes |
| "Virtue ethics gives clear rules" | Virtue ethics is deliberately flexible — it's about character, not rules |
| "Kant's Categorical Imperative is just the Golden Rule" | CI is about universalisability (could everyone do this?), not reciprocity |
| "Cultural relativism = respecting other cultures" | It's the claim that no universal morality exists — which means you can't condemn harmful practices |

### 2. Applying Theories to Scenarios

| Trap | Correct |
|------|---------|
| "Only apply one theory" | Most scenarios require analysis from MULTIPLE frameworks to show different perspectives |
| "Consequentialist analysis only considers good outcomes" | Must weigh both benefits AND harms, then calculate net utility |
| "If consequences are bad, it's always wrong" | A deontologist would say a rule-following action is right even with bad consequences |
| "Virtue ethics means 'follow your heart'" | It means developing character through practice and wisdom — not just gut feelings |

### 3. Fallacy Identification

| Trap | Correct |
|------|---------|
| "Any strong emotional language is appeal to emotion fallacy" | Only when emotion REPLACES reasoning — legitimate emotional responses aren't fallacies |
| "Citing an expert is always appeal to authority" | Only a fallacy when the authority is NOT relevant to the claim (maths prof on atheism) |
| "Correlation means false cause fallacy" | False cause = claiming causation from correlation. Noting correlation itself isn't a fallacy. |
| "All analogies are false analogies" | An analogy is a fallacy only when the comparison breaks down on relevant points |

### 4. SE Code of Ethics Mistakes

| Trap | Correct |
|------|---------|
| "If the Code doesn't forbid it, it's acceptable" | The Code shouldn't be read legalistically — judgment is required |
| "Public interest always means majority benefit" | Public interest includes protecting minorities, individual rights, not just majority |
| "Client/employer interests come first" | Public interest is PARAMOUNT — client interests are secondary (Clause 2.09) |
| "The Code gives mechanical answers" | "No mechanical process for determining if an action is right or wrong" — the Preamble says this explicitly |

### 5. Privacy & Contextual Integrity

| Trap | Correct |
|------|---------|
| "Privacy = keeping secrets" | Privacy = appropriate information flow according to contextual norms |
| "If data is technically accessible, there's no privacy" | Technical feasibility doesn't justify access (Prof Blake case) |
| "Consent = ticking a checkbox" | Meaningful consent requires understanding; dark patterns undermine it |
| "CI only applies to digital data" | CI applies to ALL information flows in any context |
| "If you shared data in one context, it's fair game" | Data shared for one purpose (delivery) cannot be used for another (marketing) — CI violation |

### 6. Intellectual Property

| Trap | Correct |
|------|---------|
| "Copyright covers ideas" | Copyright covers EXPRESSION, not ideas. Patents cover inventions/methods. |
| "Fair use means you can copy for education without limit" | Four factors must ALL be considered; substantial copying may not qualify |
| "Software can be patented everywhere" | Software patents are highly jurisdiction-dependent and often excluded |
| "Open source means no copyright" | Open source IS copyrighted — the licence grants specific freedoms |
| "GPL means you can do anything" | GPL requires derivatives to also be GPL — restricts proprietary use |

### 7. Accountability & Responsibility

| Trap | Correct |
|------|---------|
| "Responsibility and accountability are the same" | Responsibility = forward-looking (duty). Accountability = backward-looking (answering for what happened). |
| "If you didn't intend harm, you're not accountable" | Intent is ONE of Birsch's 5 criteria — causation, knowledge, capacity, and freedom also matter |
| "Junior developers can't be accountable" | Reduced accountability (capacity criterion), but NOT zero — they still have professional duties |
| "The Problem of Many Hands means nobody is responsible" | It means responsibility is DIFFICULT to assign, not that it doesn't exist |

### 8. Research Ethics

| Trap | Correct |
|------|---------|
| "Honest error is research misconduct" | NSF definition explicitly excludes honest error and differences of opinion |
| "Getting a grant means the funder owns the data" | Data ownership is complex and context-dependent; not automatic |
| "Technical contribution = authorship" | Authorship requires intellectual contribution (design, analysis, writing), not just technical execution |
| "If participants aren't harmed, ethics approval isn't needed" | Ethics approval considers risk, consent, and data handling — even low-risk studies need review |

### 9. Safety & System Failures

| Trap | Correct |
|------|---------|
| "Testing proves software is correct" | Testing shows presence of defects; CANNOT prove absence of defects |
| "Reusing code is always safe" | Ariane 5 failed because code from Ariane 4 was reused in a different context |
| "One bug is just one bug" | In networked systems, one bug can propagate (AT&T: one line crashed half the network) |
| "If users are warned, developers aren't responsible" | Warnings don't eliminate professional duty to make safe systems |
