## Cyberspace

"Cyberspace" was coined by William Gibson (1984, *Neuromancer*) — a virtual environment in which users interact using a computer network.

## Evolution of Hacking

| Phase | Period | Characterisation |
|-------|--------|-----------------|
| Phase I | 1960s–1970s | Hacker = creative programmer, "computer virtuoso," wrote elegant code |
| Phase II | 1970s–1990s | Hacking took negative meaning: breaking into systems without authorisation |
| Phase III | Mid-1990s+ | Diversification across web, mobile, e-commerce |

### Phase II Activities
- Computer viruses (1970s–1980s)
- Digital piracy, malicious cyber-behaviour (1990s)
- Identity theft, hacking personal data/devices (2000s)

## Legitimate vs Illegitimate Use

### Legitimate Use
- Use an account that is yours, valid, and legitimate
- Use consistent with established guidelines and policies

### Risk Model

```
Legitimate User → Network Connection → Computer System → Data/Code
       ↑                                       ↑
   ATTACKER → Illegitimate Access → Illegitimate Use / Malicious Code
```

## Classification of Cybercrime

### Cyber-Related Crime
Crimes where computers/internet are used in commission but are **not** the primary instrument.

| Subtype | Examples |
|---------|---------|
| **Cyber-assisted** | Tax fraud, counterfeit goods sales, forged documents |
| **Cyber-exacerbated** | Cyberstalking, cyber-pornography, cyberbullying |

### Cyber-Specific Crime
True cybercrime — can only be carried out through cybertechnology.

| Type | Description |
|------|-------------|
| **Cyber-piracy** | Unauthorised copying/distribution of digital content |
| **Cyber-trespass** | Unauthorised access to computer systems |
| **Cyber-vandalism** | Deliberate damage to digital systems/data |

### Common Attack Types

| Attack | Description |
|--------|-------------|
| IP Spoofing | Forging source IP address to impersonate |
| Virus | Malicious code that attaches to programs |
| Worm | Self-replicating malware that spreads across networks |
| Denial of Service (DoS) | Overwhelming system to prevent legitimate use |
| Cyber-fraud | Online scams and deception for financial gain |
| Phishing | Deceptive emails/sites to steal credentials |
| Ransomware | Encrypting victim's data, demanding payment |

## Ethical Analysis of Cyber-Behaviour

### Using the SE Code of Ethics

Cyber-criminals violate all eight principles of the SE Code:
1. **Public Interest**: Harm to society
2. **Client & Employer**: Breach of trust
3. **Product**: Subverting systems
4. **Judgment**: Self-interest over ethics
5. **Management**: No ethical oversight
6. **Profession**: Damaging reputation of computing
7. **Colleagues**: Undermining others' work
8. **Self**: No commitment to ethical development

### Key Ethical Questions

| Question | Why it matters |
|----------|---------------|
| What activities are criminal? | Defines legal boundaries |
| Why are they criminal? | Identifies the harm principle |
| Who is affected? | Maps stakeholders |
| Who is doing them? | Identifies responsible parties |
| How to protect against them? | Duty of computing professionals |

## Ethical Tensions in Cybersecurity

| Tension | Competing Values |
|---------|-----------------|
| Security research vs hacking | Knowledge advancement vs potential for harm |
| Vulnerability disclosure | Public safety vs giving attackers tools |
| Encryption vs law enforcement | Individual privacy vs public safety |
| Surveillance vs freedom | Security vs civil liberties |
| "Ethical hacking" | Testing defences vs unauthorised access |
