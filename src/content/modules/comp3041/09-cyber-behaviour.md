## Cyberspace

:::eli10

Cyberspace is the virtual world inside computers and the internet. It is not a physical place, but people meet, talk, play, and work there. Just like the real world has rules and crimes, cyberspace does too.

:::

:::eli15
Cyberspace is the virtual environment created by interconnected computer networks where people interact, communicate, and transact. The term was coined by William Gibson in 1984. Though not a physical space, cyberspace is governed by social norms, laws, and ethical obligations. Activities in cyberspace have real-world consequences, making the ethical rules that apply online just as important as those offline.

:::

:::eli20
"Cyberspace" was coined by William Gibson (1984, *Neuromancer*) — a virtual environment in which users interact using a computer network.

:::

## Evolution of Hacking

:::eli10

In the early days, "hacker" meant someone really clever with computers who wrote amazing code. Over time, the word changed to mean someone who breaks into computers without permission. Now there are many types of computer crimes, from stealing identities to spreading viruses.

:::

:::eli15
The meaning of "hacking" has evolved through three phases. In the 1960s-70s, hackers were creative programmers admired for elegant code. By the 1970s-90s, the term shifted to mean unauthorised system intrusion, alongside the rise of viruses and digital piracy. From the mid-1990s onward, cybercrime diversified across web, mobile, and e-commerce platforms to include identity theft, ransomware, and large-scale data breaches.

:::

:::eli20
| Phase | Period | Characterisation |
|-------|--------|-----------------|
| Phase I | 1960s–1970s | Hacker = creative programmer, "computer virtuoso," wrote elegant code |
| Phase II | 1970s–1990s | Hacking took negative meaning: breaking into systems without authorisation |
| Phase III | Mid-1990s+ | Diversification across web, mobile, e-commerce |

### Phase II Activities
- Computer viruses (1970s–1980s)
- Digital piracy, malicious cyber-behaviour (1990s)
- Identity theft, hacking personal data/devices (2000s)

:::

## Legitimate vs Illegitimate Use

:::eli10

Using a computer legitimately means using your own account, following the rules, and not doing anything you are not allowed to do. Illegitimate use means breaking in, pretending to be someone else, or using a system to do harm. It is like the difference between walking through an open door and breaking a window to get in.

:::

:::eli15
Legitimate computer use means using valid accounts consistent with established policies and guidelines. Illegitimate use involves unauthorised access (gaining entry without permission), illegitimate use of access (using authorised access for unauthorised purposes), or deploying malicious code. The risk model shows how attackers exploit network connections to access systems and data they should not, contrasted with legitimate users following proper channels.

:::

:::eli20
### Legitimate Use
- Use an account that is yours, valid, and legitimate
- Use consistent with established guidelines and policies

### Risk Model

```
Legitimate User → Network Connection → Computer System → Data/Code
       ↑                                       ↑
   ATTACKER → Illegitimate Access → Illegitimate Use / Malicious Code
```

:::

## Classification of Cybercrime

:::eli10

There are two types of computer crime. The first type uses computers to help commit a crime that could also happen without computers (like fraud or bullying, but done online). The second type can only happen with computers (like hacking into a system, spreading computer viruses, or pirating digital music).

:::

:::eli15
Cybercrime is classified into two broad categories. Cyber-related crime uses computers as a tool but the crime could theoretically occur without technology (tax fraud done online, cyberbullying). Cyber-specific crime can only exist through technology: cyber-piracy (unauthorised copying of digital content), cyber-trespass (unauthorised access), and cyber-vandalism (deliberate destruction of digital systems). Common attack vectors include IP spoofing, viruses, worms, denial-of-service attacks, phishing, and ransomware.

:::

:::eli20
### Cyber-Related Crime
Crimes where computers/internet are used in commission but are **not** the primary instrument.

| Subtype | Examples |
|---------|---------|
| **Cyber-assisted** | Tax fraud, counterfeit goods sales, forged documents |
| **Cyber-exacerbated** | Cyberstalking, cyber-pornography, cyberbullying |

### Cyber-Specific Crime
True cybercrime — can only be carried out through cybertechnology.

| Type | Description |
|------|-------------|
| **Cyber-piracy** | Unauthorised copying/distribution of digital content |
| **Cyber-trespass** | Unauthorised access to computer systems |
| **Cyber-vandalism** | Deliberate damage to digital systems/data |

### Common Attack Types

| Attack | Description |
|--------|-------------|
| IP Spoofing | Forging source IP address to impersonate |
| Virus | Malicious code that attaches to programs |
| Worm | Self-replicating malware that spreads across networks |
| Denial of Service (DoS) | Overwhelming system to prevent legitimate use |
| Cyber-fraud | Online scams and deception for financial gain |
| Phishing | Deceptive emails/sites to steal credentials |
| Ransomware | Encrypting victim's data, demanding payment |

:::

## Ethical Analysis of Cyber-Behaviour

:::eli10

When people commit cybercrimes, they break every rule that professional computing people are supposed to follow. They hurt society, break trust, damage the reputation of computing, and do not care about doing the right thing. The ethical codes exist so that professionals know where the line is.

:::

:::eli15
Cybercriminals violate all eight principles of the Software Engineering Code of Ethics: they harm the public interest, breach trust with clients and employers, subvert products, prioritise self-interest over ethical judgment, operate without oversight, damage the profession's reputation, undermine colleagues' work, and show no commitment to ethical growth. Analysing cybercrime through professional codes helps clarify why these activities are wrong beyond just being illegal.

:::

:::eli20
### Using the SE Code of Ethics

Cyber-criminals violate all eight principles of the SE Code:
1. **Public Interest**: Harm to society
2. **Client & Employer**: Breach of trust
3. **Product**: Subverting systems
4. **Judgment**: Self-interest over ethics
5. **Management**: No ethical oversight
6. **Profession**: Damaging reputation of computing
7. **Colleagues**: Undermining others' work
8. **Self**: No commitment to ethical development

### Key Ethical Questions

| Question | Why it matters |
|----------|---------------|
| What activities are criminal? | Defines legal boundaries |
| Why are they criminal? | Identifies the harm principle |
| Who is affected? | Maps stakeholders |
| Who is doing them? | Identifies responsible parties |
| How to protect against them? | Duty of computing professionals |

:::

## Ethical Tensions in Cybersecurity

:::eli10

Sometimes good goals fight each other in cybersecurity. Security researchers need to find weaknesses in systems to fix them, but the same knowledge could help criminals. Governments want to catch bad guys by watching everyone, but that means watching innocent people too. These are hard problems with no easy answers.

:::

:::eli15
Cybersecurity involves fundamental ethical tensions: security research advances knowledge but could enable harm; vulnerability disclosure protects the public but arms attackers; strong encryption protects privacy but hinders law enforcement; surveillance enhances security but erodes civil liberties; and "ethical hacking" tests defences but involves techniques identical to criminal intrusion. Professionals must navigate these competing values carefully, with no simple resolution.

:::

:::eli20
| Tension | Competing Values |
|---------|-----------------|
| Security research vs hacking | Knowledge advancement vs potential for harm |
| Vulnerability disclosure | Public safety vs giving attackers tools |
| Encryption vs law enforcement | Individual privacy vs public safety |
| Surveillance vs freedom | Security vs civil liberties |
| "Ethical hacking" | Testing defences vs unauthorised access |

:::
