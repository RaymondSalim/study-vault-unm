# Prerequisites

## Required Background Knowledge

| Area | Topics Needed | Why |
|------|---------------|-----|
| Software Development Basics | SDLC phases, basic programming experience | Understanding where requirements fit in the development process |
| UML Basics | Class diagrams, sequence diagrams (basic) | Modelling techniques used in analysis |
| Communication Skills | Clear writing, structured thinking | Requirements must be written precisely and unambiguously |
| Basic Project Management | Stakeholder concept, project lifecycle | Understanding who requirements serve and how they are managed |

## Helpful Prior Modules

- **COMP1011 - Computer Fundamentals** -- General understanding of how software systems work
- **COMP1012 - Programming** -- Practical understanding of what can and cannot be implemented
- **COMP2019 - Software Engineering Group Project** -- Hands-on experience with working in teams and delivering software
- **Any group project experience** -- Understanding stakeholder needs and communication challenges

## Key Concepts You Should Already Know

1. What the software development lifecycle (SDLC) is and its main phases
2. Basic UML notation (at minimum, what a class diagram looks like)
3. The difference between a customer/user and a developer
4. What "scope" means in a project context
5. How to write clearly and unambiguously in English

## Self-Assessment Test

Answer these before starting the module. If you struggle with more than 2, review the prerequisites first.

| # | Question | Expected Answer |
|---|----------|-----------------|
| 1 | Name the main phases of the software development lifecycle. | Requirements, Design, Implementation, Testing, Deployment, Maintenance |
| 2 | What is a stakeholder in a software project? | Anyone who has an interest in or is affected by the system (users, clients, developers, managers) |
| 3 | What is the difference between waterfall and agile development? | Waterfall is sequential (each phase completes before the next); agile is iterative with frequent feedback and incremental delivery |
| 4 | Rewrite "The system should be fast" to be more specific. | "The system shall respond to user queries within 2 seconds under normal load (100 concurrent users)" |
| 5 | What does UML stand for? | Unified Modeling Language |
