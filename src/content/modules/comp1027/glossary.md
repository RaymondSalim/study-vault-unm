## A-D

| Term | Definition |
|------|-----------|
| **Accumulator** | Register that stores the result of ALU operations |
| **Address bus** | Unidirectional bus carrying memory addresses from CPU to memory |
| **ALU** | Arithmetic Logic Unit; performs arithmetic and logical operations |
| **ARP** | Address Resolution Protocol; maps IP addresses to MAC addresses |
| **Assembly language** | Low-level language using mnemonics for machine instructions |
| **Bandwidth** | Maximum data transfer rate of a network link |
| **Binary** | Base-2 number system using digits 0 and 1 |
| **Boolean algebra** | Mathematical system for logic with values TRUE/FALSE (1/0) |
| **Broadcast address** | Network address where all host bits are 1; reaches all hosts |
| **Bus** | Shared communication pathway between components |
| **Cache** | Fast small memory between CPU and RAM for frequently used data |
| **CIDR** | Classless Inter-Domain Routing; flexible subnet notation (e.g., /24) |
| **Combinational circuit** | Circuit whose output depends only on current inputs |
| **Context switch** | Saving state of one process and loading another |
| **Control unit** | CPU component that decodes instructions and generates control signals |
| **Counter** | Sequential circuit that cycles through a sequence of states |
| **Data bus** | Bidirectional bus carrying data between CPU and memory |
| **De Morgan's theorem** | (AB)' = A'+B' and (A+B)' = A'B' |
| **Decoder** | Circuit with n inputs producing 2^n outputs (one-hot) |
| **DHCP** | Dynamic Host Configuration Protocol; auto-assigns IP addresses |
| **DNS** | Domain Name System; translates domain names to IP addresses |
| **DRAM** | Dynamic RAM; uses capacitors, needs refresh, cheaper than SRAM |

## E-I

| Term | Definition |
|------|-----------|
| **Encoder** | Circuit with 2^n inputs producing n-bit output code |
| **Ethernet** | Layer 2 LAN protocol using MAC addresses and CSMA/CD |
| **Fetch-Decode-Execute** | CPU cycle: get instruction, interpret it, perform operation |
| **FIFO** | First In First Out; scheduling/replacement policy |
| **Firewall** | Network security device that filters traffic based on rules |
| **Flip-flop** | Edge-triggered 1-bit memory element |
| **Frame** | Fixed-size block of physical memory (paging) |
| **Full adder** | Adds two bits plus carry-in; produces sum and carry-out |
| **Gateway** | Device connecting networks using different protocols |
| **Half adder** | Adds two bits; produces sum and carry (no carry-in) |
| **Harvard architecture** | Separate memory and buses for instructions and data |
| **Hexadecimal** | Base-16 number system (0-9, A-F) |
| **HTTP** | HyperText Transfer Protocol; web page transfer (port 80) |
| **Hub** | Layer 1 device that broadcasts all traffic to all ports |
| **Instruction register (IR)** | Holds the instruction currently being executed |
| **Interrupt** | Signal that causes CPU to suspend current task and handle event |
| **IP address** | Logical 32-bit (IPv4) or 128-bit (IPv6) network address |

## J-P

| Term | Definition |
|------|-----------|
| **JK flip-flop** | Flip-flop with Set, Reset, Hold, and Toggle modes |
| **Karnaugh map** | Visual method for simplifying Boolean expressions |
| **Kernel** | Core OS code running in privileged mode |
| **Latch** | Level-sensitive 1-bit memory element |
| **LRU** | Least Recently Used; page replacement algorithm |
| **MAC address** | 48-bit physical hardware address of a network interface |
| **MAR** | Memory Address Register; holds address for memory access |
| **MDR** | Memory Data Register; holds data being transferred to/from memory |
| **Minterm** | Product term with all variables present (SOP building block) |
| **Multiplexer (MUX)** | Selects one of many inputs based on select lines |
| **NAND gate** | Universal gate; output 0 only when both inputs are 1 |
| **NOR gate** | Universal gate; output 1 only when both inputs are 0 |
| **Opcode** | Portion of instruction specifying the operation |
| **OSI model** | 7-layer reference model for network communication |
| **Page** | Fixed-size block of logical/virtual memory |
| **Page fault** | Interrupt when requested page is not in RAM |
| **Paging** | Memory management dividing logical memory into fixed-size pages |
| **Packet** | Layer 3 PDU containing source/destination IP and data |
| **PC (Program Counter)** | Register holding address of next instruction to fetch |
| **Process** | Program in execution with its own memory space and state |
| **Protocol** | Set of rules governing communication between devices |

## Q-Z

| Term | Definition |
|------|-----------|
| **RAM** | Random Access Memory; volatile read/write storage |
| **Register** | Small, fast storage location inside the CPU |
| **Ripple counter** | Asynchronous counter where each FF is clocked by previous FF |
| **ROM** | Read-Only Memory; non-volatile, stores firmware/instructions |
| **Router** | Layer 3 device that forwards packets between networks using IP |
| **Round Robin** | Scheduling algorithm giving each process equal time quantum |
| **Scheduling** | OS deciding which process runs next on the CPU |
| **Sequential circuit** | Circuit whose output depends on current inputs AND stored state |
| **Shift register** | Register that shifts bits left or right each clock cycle |
| **SRAM** | Static RAM; uses flip-flops, no refresh needed, faster/costlier |
| **Subnet mask** | Bit mask separating network and host portions of IP address |
| **Switch** | Layer 2 device that forwards frames based on MAC address |
| **TCP** | Transmission Control Protocol; reliable, connection-oriented |
| **Truth table** | Table listing all input combinations and corresponding outputs |
| **Two's complement** | Binary encoding for signed integers; invert + 1 for negation |
| **UDP** | User Datagram Protocol; fast, connectionless, unreliable |
| **Virtual memory** | Technique using disk to extend apparent RAM size |
| **Von Neumann architecture** | Single memory for both data and instructions, shared bus |
| **XOR gate** | Output 1 when inputs differ; used in parity and addition |
