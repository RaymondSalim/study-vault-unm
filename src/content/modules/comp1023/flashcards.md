## Key Concept Flashcards

| # | Question | Answer |
|---|----------|--------|
| 1 | What is the SDLC? | The Software Development Life Cycle — a structured process for planning, creating, testing, and deploying software. |
| 2 | Name the phases of the Waterfall model. | Requirements, Design, Implementation, Verification, Maintenance — each phase completes before the next begins. |
| 3 | What is Agile? | An iterative approach to software development that emphasises collaboration, working software, and responding to change over following a rigid plan. |
| 4 | What is a user story? | A short description of a feature from the user's perspective: "As a [role], I want [feature] so that [benefit]." |
| 5 | What is a use case? | A description of how a user interacts with a system to achieve a specific goal, including actors, preconditions, and flow of events. |
| 6 | What is the difference between functional and non-functional requirements? | Functional requirements define what the system does; non-functional requirements define how well it does it (performance, security, usability). |
| 7 | What is a UML class diagram? | A structural diagram showing classes, their attributes, methods, and relationships (association, inheritance, composition). |
| 8 | What does multiplicity mean in UML? | It specifies how many instances of one class can be associated with instances of another (e.g., 1..*, 0..1). |
| 9 | What is a sequence diagram? | A UML behavioural diagram showing how objects interact over time through message passing. |
| 10 | What is the difference between aggregation and composition? | Aggregation is a "has-a" relationship where parts can exist independently; composition is a stronger "owns-a" where parts are destroyed with the whole. |
| 11 | What are the four values of the Agile Manifesto? | Individuals over processes, working software over documentation, customer collaboration over contracts, responding to change over following a plan. |
| 12 | What is a sprint in Scrum? | A fixed time-box (usually 2-4 weeks) during which a potentially shippable product increment is created. |
| 13 | What is unit testing? | Testing individual components or methods in isolation to verify they produce correct output for given inputs. |
| 14 | What is integration testing? | Testing how multiple components work together to ensure interfaces and interactions function correctly. |
| 15 | What is the Open/Closed Principle? | Software entities should be open for extension but closed for modification. |
| 16 | What is coupling? | The degree of interdependence between modules; low coupling is preferred for maintainability. |
| 17 | What is cohesion? | The degree to which elements within a module belong together; high cohesion is preferred. |
| 18 | What is refactoring? | Restructuring existing code without changing its external behaviour to improve readability and maintainability. |
| 19 | What is the MoSCoW method? | A prioritisation technique: Must have, Should have, Could have, Won't have (this time). |
| 20 | What is a design pattern? | A reusable solution to a commonly occurring problem in software design (e.g., Singleton, Observer, Factory). |
| 21 | What is version control? | A system that records changes to files over time so you can recall specific versions later (e.g., Git). |
| 22 | What is the purpose of a Software Requirements Specification (SRS)? | To document all functional and non-functional requirements in a clear, unambiguous way for stakeholders and developers. |
| 23 | What is black-box testing? | Testing based on specifications without knowledge of internal implementation. |
| 24 | What is white-box testing? | Testing based on knowledge of internal code structure, including paths and branches. |
| 25 | What is the Single Responsibility Principle? | A class should have only one reason to change — it should do one thing and do it well. |
