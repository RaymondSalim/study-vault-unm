## What You Need to Know Before Starting

### Essential Prior Knowledge

- **Basic programming:** Understanding of variables, loops, conditionals, and functions in any language
- **Object-Oriented concepts:** Familiarity with classes, objects, methods, and attributes
- **General problem-solving:** Ability to break a large problem into smaller parts
- **Basic communication skills:** Writing clear descriptions and reading specifications

### Helpful But Not Required

- Experience working in a team on a coding project
- Familiarity with version control (Git)
- Any exposure to project management or planning tools
- Experience reading or writing documentation

### Prior Modules That Help

- **COMP1028 - Programming and Algorithms:** Provides the OOP knowledge needed for understanding class diagrams and design principles
- Any introductory programming module that covers classes and inheritance

## Quick Self-Test

Answer these 5 questions to check your readiness:

1. **OOP Basics:** What is the difference between a class and an object?
   - *Expected answer: A class is a blueprint/template; an object is an instance of a class.*

2. **Inheritance:** If class Dog extends class Animal, what does Dog inherit?
   - *Expected answer: All public/protected attributes and methods of Animal.*

3. **Problem decomposition:** If asked to build a library system, name 3 classes you might need.
   - *Expected answer: e.g., Book, Member, Loan (or similar reasonable decomposition)*

4. **Requirements:** What is the difference between "The system shall allow login" and "The system shall respond within 2 seconds"?
   - *Expected answer: First is functional (what it does), second is non-functional (how well it performs).*

5. **Collaboration:** Name one benefit of using version control in a team project.
   - *Expected answer: e.g., Track changes, enable parallel work, revert mistakes, maintain history.*

**Scoring:** If you got 4-5 correct, you are well prepared. If fewer than 3, revise basic OOP concepts and consider how large systems are structured.
