## Time Allocation

- **Total exam time:** Typically 2 hours
- **Reading time:** First 5 minutes — scan all questions and plan which ones to attempt first
- **Short answer questions:** 3-5 minutes each
- **Diagram questions (UML):** 10-15 minutes each — draw neatly with proper notation
- **Essay/discussion questions:** 15-20 minutes each
- **Buffer:** 10 minutes at the end for review

## Topic Weighting

| Topic | Approximate Weight | Priority |
|-------|-------------------|----------|
| SDLC and Processes (Waterfall, Agile, Scrum) | 20% | High |
| Requirements Engineering | 15% | High |

| UML Class Diagrams | 20% | High |
| UML Sequence Diagrams | 10% | Medium |
| OOP Design Principles | 15% | High |
| Testing | 10% | Medium |

## Question Types to Expect

- **Diagram drawing:** Draw a class diagram / sequence diagram for a given scenario
- **Short answer:** Define terms, explain differences (e.g., aggregation vs composition)
- **Scenario analysis:** Given a description, identify requirements, actors, or design issues
- **Essay/discussion:** Compare methodologies, argue for a development approach
- **True/False with justification:** Identify whether design principles are violated

## Key Definitions to Memorise

1. Functional vs Non-functional requirements
2. Coupling (low = good) and Cohesion (high = good)
3. SOLID principles (especially SRP and OCP)
4. UML notation: associations, multiplicities, inheritance arrows, composition diamonds
5. Agile Manifesto four values and twelve principles
6. Testing types: unit, integration, system, acceptance
7. Use case components: actor, precondition, main flow, alternative flows, postcondition
8. Waterfall vs Agile trade-offs

## Night Before Checklist

1. ☐ Can you draw a correct UML class diagram with proper notation (arrows, diamonds, multiplicities)?
2. ☐ Do you know the difference between aggregation (hollow diamond) and composition (filled diamond)?
3. ☐ Can you write a complete use case with actors, preconditions, and flows?
4. ☐ Can you explain Agile vs Waterfall with advantages and disadvantages of each?
5. ☐ Do you know all SOLID principles and can give an example of each?
6. ☐ Can you draw a sequence diagram with lifelines, messages, and return values?
7. ☐ Can you explain different testing levels and when each is used?
8. ☐ Do you understand requirements elicitation techniques (interviews, surveys, observation)?
9. ☐ Can you identify coupling/cohesion issues in a given code scenario?
10. ☐ Do you know the Scrum roles (Product Owner, Scrum Master, Development Team) and ceremonies?
