## Lecture Slides Overview

| Lecture | Topic | Key Concepts | Exam Weight |
|---------|-------|--------------|-------------|
| 1-2 | Functional Basics (Haskell) | Pure functions, immutability, referential transparency, basic syntax, guards, where/let, pattern matching, if-then-else | 15% |
| 3-4 | Lists and Recursion (Haskell) | List construction (cons `:`, `[]`), head/tail, pattern matching on lists, recursive list functions (length, sum, reverse, map-like) | 20% |
| 5-6 | Higher-Order Functions | map, filter, foldr/foldl, function composition (.), lambda expressions, partial application, currying, sections | 15% |
| 7-8 | Types and Type Classes | Type signatures, polymorphism (type variables), type inference, Eq, Ord, Show, Num, custom data types, deriving | 10% |
| 9-10 | Prolog Basics | Facts, rules, queries, variables, atoms, unification, backtracking, trace, cut (!), negation as failure | 15% |
| 11-12 | Prolog Lists and Recursion | List notation [H|T], member/2, append/3, reverse, length, list processing, accumulators, generate-and-test | 15% |
| 13-14 | Paradigm Comparison | Declarative vs imperative, functional vs logic vs OOP, evaluation strategies (lazy vs strict), trade-offs, when to use which paradigm | 10% |

## Notes

- Haskell content (Lectures 1-8) makes up ~60% of the exam — master this first.
- Lists and Recursion is the single most important topic for both Haskell and Prolog.
- Higher-Order Functions questions often ask you to rewrite explicit recursion using map/filter/fold.
- Prolog query tracing (showing unification and backtracking steps) is a very common exam question format.
- The Paradigm Comparison section often appears as an essay question — prepare structured arguments.
- Type inference questions require you to work out the type of composed/partially applied functions.
