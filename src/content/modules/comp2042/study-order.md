# Study Order

| # | Topic | Why this order | Estimated time |
|---|-------|---------------|----------------|
| 1 | Maintenance Challenges | Understand WHY maintainability matters before learning HOW to achieve it | 2 hours |
| 2 | OOP Design Principles (SOLID) | Foundation for all patterns and refactoring decisions | 3 hours |
| 3 | Creational Design Patterns | Simplest patterns; builds on OOP principles just learned | 2.5 hours |
| 4 | Structural Design Patterns | Extends understanding of composition; builds on creational patterns | 2.5 hours |
| 5 | Behavioural Design Patterns | Most complex patterns; requires solid understanding of OOP and structure | 3 hours |
| 6 | Refactoring & Code Smells | Applying principles and patterns to improve existing code; needs pattern knowledge | 3 hours |
| 7 | Testing & Build Tools | Testing validates refactoring; build tools support the development workflow | 2.5 hours |
| 8 | GUI & Maintenance | Applies MVC pattern and maintenance principles to a concrete domain | 2 hours |

## Dependency Rationale

The order builds from "why" to "principles" to "patterns" to "practice":

1. **Motivation** (Maintenance Challenges) -- understand the problem
2. **Principles** (SOLID) -- learn the rules that guide good design
3. **Solutions** (Design Patterns) -- concrete reusable solutions applying those principles
4. **Improvement** (Refactoring) -- fix existing code using principles and patterns
5. **Verification** (Testing) -- ensure refactoring doesn't break anything
6. **Application** (GUI) -- real-world application of all concepts together
