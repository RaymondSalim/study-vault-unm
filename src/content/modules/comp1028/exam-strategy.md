## Time Allocation

- **Total exam time:** Typically 2 hours
- **Reading time:** First 5 minutes — identify code-tracing vs code-writing questions
- **Code tracing questions:** 5-10 minutes each (follow execution carefully line by line)
- **Code writing questions:** 10-20 minutes each (plan structure before writing)
- **Theory/short answer:** 3-5 minutes each
- **Buffer:** 10 minutes to review code for off-by-one errors and missing return statements

## Question Types to Expect

- **Code tracing:** Given code, determine the output (watch for recursion and polymorphism)
- **Code writing:** Write a method/class to solve a problem (e.g., recursive method, sorting implementation)
- **Error identification:** Find and fix bugs in given code
- **OOP design:** Design a class hierarchy for a given scenario
- **Algorithm analysis:** Determine time complexity using Big-O notation
- **Short answer:** Explain concepts (inheritance vs composition, abstract vs interface)

## Key Things to Memorise

1. Array declaration, initialization, and traversal patterns
2. The difference between `==` and `.equals()` for objects
3. Constructor chaining with `super()` and `this()`
4. Method overriding rules (same signature, covariant return type)
5. Recursion pattern: base case + recursive case
6. Binary search requires sorted input and uses O(log n)
7. Sorting complexities: Bubble/Selection/Insertion = O(n^2), Merge Sort = O(n log n)
8. Abstract class vs Interface differences
9. try-catch-finally execution flow
10. ArrayList methods: add(), get(), remove(), size()

## Night Before Checklist

1. ☐ Can you trace through recursive code and determine the output?
2. ☐ Can you write a binary search from scratch?
3. ☐ Do you understand dynamic dispatch (which overridden method gets called)?
4. ☐ Can you implement a class hierarchy with inheritance and method overriding?
5. ☐ Can you write merge sort or explain its divide-and-conquer strategy?
6. ☐ Do you know when to use abstract class vs interface?
7. ☐ Can you identify the Big-O complexity of a given code snippet?
8. ☐ Can you handle exceptions with try-catch and explain checked vs unchecked?
9. ☐ Do you understand how ArrayList differs from arrays?
10. ☐ Can you write a complete Java class with constructors, getters/setters, and toString()?
