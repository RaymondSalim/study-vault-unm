## Lecture Slides Overview

| Lecture | Topic | Key Concepts | Exam Weight |
|---------|-------|--------------|-------------|
| 1-2 | Java Basics | Variables, data types (int, double, boolean, char, String), operators, input/output, if/else, switch, type casting | 10% |
| 3-4 | Arrays and Methods | Array declaration/traversal, for/while loops, method definition, parameters, return types, method overloading, scope | 15% |
| 5-6 | OOP Basics | Classes, objects, constructors, encapsulation (private fields, getters/setters), `this` keyword, toString(), static vs instance | 15% |
| 7-8 | Inheritance and Polymorphism | extends, super, method overriding, dynamic dispatch, abstract classes, upcasting/downcasting, Object class | 20% |
| 9-10 | Interfaces and Collections | Interface definition, implements, multiple interfaces, ArrayList, HashMap, Iterator, Comparable, generics basics | 10% |
| 11-12 | Recursion | Base case and recursive case, call stack, recursive vs iterative, recursion on arrays, recursion on strings, stack overflow | 15% |
| 13-14 | Searching and Sorting | Linear search, binary search, bubble sort, selection sort, insertion sort, merge sort, Big-O analysis, algorithm comparison | 15% |
| 15 | Exceptions and I/O | try-catch-finally, checked vs unchecked exceptions, throwing exceptions, file reading/writing, Scanner, BufferedReader | (included in coursework) |

## Notes

- Inheritance and Polymorphism is the single most weighted topic (~20%) — expect code tracing with dynamic dispatch.
- Recursion and Sorting together make up ~30% — practise writing these algorithms from scratch on paper.
- OOP design questions may combine multiple topics (inheritance + interfaces + collections).
- Code tracing questions are common — follow execution line by line, track variable values carefully.
- Exceptions/I/O may be more relevant to coursework than the written exam, but still review the basics.
