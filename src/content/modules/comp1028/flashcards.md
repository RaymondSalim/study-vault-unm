## Key Concept Flashcards

| # | Question | Answer |
|---|----------|--------|
| 1 | What is a variable in Java? | A named storage location in memory that holds a value of a specific data type. |
| 2 | What is the difference between primitive and reference types? | Primitives (int, double, boolean, char) store values directly; reference types (objects, arrays) store memory addresses pointing to data. |
| 3 | What is an array? | A fixed-size, indexed collection of elements of the same type, accessed by position (0-based indexing). |
| 4 | What is the difference between a while loop and a for loop? | A for loop is used when the number of iterations is known; a while loop is used when the termination condition is checked dynamically. |
| 5 | What is a method? | A named block of code that performs a specific task, can accept parameters, and may return a value. |
| 6 | What is method overloading? | Defining multiple methods with the same name but different parameter lists (different types or number of parameters). |
| 7 | What is a class? | A blueprint that defines the attributes (fields) and behaviours (methods) that objects of that type will have. |
| 8 | What is encapsulation? | Hiding internal state and requiring all interaction through an object's methods; achieved using private fields and public getters/setters. |
| 9 | What is inheritance? | A mechanism where a subclass acquires the fields and methods of a superclass, enabling code reuse and hierarchical relationships. |
| 10 | What is polymorphism? | The ability of objects of different classes to respond to the same method call in different ways (method overriding). |
| 11 | What is an abstract class? | A class that cannot be instantiated and may contain abstract methods (no body) that subclasses must implement. |
| 12 | What is an interface in Java? | A contract that defines method signatures without implementations; a class can implement multiple interfaces. |
| 13 | What is the difference between ArrayList and array? | ArrayList is dynamically resizable and holds objects; arrays are fixed-size and can hold primitives or objects. |
| 14 | What is recursion? | A technique where a method calls itself with a simpler input, with a base case to stop the recursion. |
| 15 | What are the two essential parts of a recursive method? | Base case (termination condition) and recursive case (the method calls itself with a smaller problem). |
| 16 | What is the time complexity of linear search? | O(n) — in the worst case, every element must be checked. |
| 17 | What is the time complexity of binary search? | O(log n) — the search space is halved each iteration; requires a sorted array. |
| 18 | What is the time complexity of bubble sort? | O(n^2) — nested loops compare and swap adjacent elements. |
| 19 | How does merge sort work? | Divide the array in half recursively until single elements, then merge sorted halves back together. O(n log n) time. |
| 20 | What is the difference between checked and unchecked exceptions? | Checked exceptions must be caught or declared (e.g., IOException); unchecked exceptions (RuntimeException subclasses) do not require explicit handling. |
| 21 | What is a try-catch block used for? | Handling exceptions gracefully — code in try may throw an exception, which is caught and handled in catch. |
| 22 | What does the `static` keyword mean? | The member belongs to the class itself rather than to instances; shared across all objects of that class. |
| 23 | What is the `this` keyword? | A reference to the current object instance, used to distinguish instance variables from parameters with the same name. |
| 24 | What is Big-O notation? | A way to describe the upper bound of an algorithm's time or space complexity as input size grows. |
| 25 | What is the Collections framework? | A set of Java interfaces and classes (List, Set, Map, etc.) that provide standard data structures and algorithms. |
