# Flashcards

| # | Question | Answer |
|---|----------|--------|
| 1 | What is the difference between a process and a thread? | A process is an independent execution unit with its own address space; a thread is a lightweight unit within a process that shares the same address space but has its own stack and registers. |
| 2 | Name the four Coffman conditions for deadlock. | Mutual exclusion, hold and wait, no preemption, circular wait. |
| 3 | What is the difference between preemptive and non-preemptive scheduling? | Preemptive scheduling can interrupt a running process to allocate CPU to another; non-preemptive scheduling lets a process run until it voluntarily yields or completes. |
| 4 | What does the Banker's Algorithm do? | It determines whether granting a resource request will leave the system in a safe state (i.e., there exists at least one sequence in which all processes can finish). |
| 5 | Explain internal vs external fragmentation. | Internal fragmentation is wasted space within an allocated block; external fragmentation is wasted space between allocated blocks that is too small to use. |
| 6 | What is a page fault? | A page fault occurs when a process accesses a page not currently in physical memory, triggering the OS to load it from disk. |
| 7 | Describe Round Robin scheduling. | Each process gets a fixed time quantum; if it doesn't finish, it's preempted and placed at the back of the ready queue. |
| 8 | What is thrashing? | Thrashing occurs when the system spends more time swapping pages in and out than executing processes, typically due to insufficient frames. |
| 9 | What is a semaphore? | A synchronisation primitive consisting of an integer variable and two atomic operations (wait/P and signal/V) used to control access to shared resources. |
| 10 | Difference between mutex and semaphore? | A mutex is a binary lock with ownership (only the locker can unlock); a semaphore is a counter that can allow multiple accesses and has no ownership concept. |
| 11 | What is the convoy effect? | In FCFS scheduling, short processes queue behind a long CPU-burst process, leading to high average waiting times. |
| 12 | Explain Belady's anomaly. | With FIFO page replacement, increasing the number of frames can paradoxically increase page faults for certain reference strings. |
| 13 | What is a critical section? | A code segment where shared resources are accessed; only one process/thread should execute in it at a time. |
| 14 | What are the three requirements for a valid critical section solution? | Mutual exclusion, progress (no unnecessary blocking), and bounded waiting (no starvation). |
| 15 | Describe the working set model. | It defines the set of pages a process has referenced within a recent time window; the OS keeps these pages in memory to reduce page faults. |
| 16 | What is a context switch? | The OS saves the state (registers, PC, etc.) of the current process and loads the state of the next process to run. |
| 17 | Explain demand paging. | Pages are loaded into memory only when accessed (on demand), not all at once when a process starts. |
| 18 | What is priority inversion? | A high-priority process is indirectly blocked by a low-priority process holding a needed resource, while a medium-priority process preempts the low-priority one. |
| 19 | What is the difference between logical and physical addresses? | Logical (virtual) addresses are generated by the CPU during execution; physical addresses refer to actual locations in RAM. The MMU translates between them. |
| 20 | Describe the LRU page replacement algorithm. | Least Recently Used replaces the page that has not been accessed for the longest time; it approximates optimal replacement. |
| 21 | What is a TLB? | The Translation Lookaside Buffer is a hardware cache of recent page table entries that speeds up virtual-to-physical address translation. |
| 22 | What is starvation and how does it differ from deadlock? | Starvation is when a process waits indefinitely because others keep getting priority; deadlock is when processes are mutually blocked and none can proceed. |
| 23 | Explain the difference between SJF and SRTF. | SJF (Shortest Job First) is non-preemptive -- once started, a job runs to completion. SRTF (Shortest Remaining Time First) is preemptive -- it switches if a new job with shorter remaining time arrives. |
| 24 | What is an inode? | A data structure in Unix file systems storing metadata about a file (permissions, size, pointers to data blocks) but not the filename. |
| 25 | What is the purpose of disk scheduling algorithms? | To minimise seek time by ordering disk I/O requests efficiently (e.g., FCFS, SSTF, SCAN, C-SCAN). |
