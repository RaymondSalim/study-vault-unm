## Question 1: Features, Image Segmentation, and Object Detection (20 marks)

### (a) K-Means for Image Segmentation

#### (i) Definitions in the K-Means Objective [3 marks]

Given the objective function:

L = Sigma_i Sigma_k r_ik ||x_i - mu_k||^2

Where:
- **x_i**: The feature vector for pixel i. In this context, it represents the measurable property extracted from each pixel (e.g., the RGB colour value, intensity, or any chosen feature descriptor).
- **mu_k**: The cluster centre (prototype/mean) for cluster k. It is the representative feature vector for the k-th segment, computed as the mean of all feature vectors assigned to cluster k.
- **N**: The total number of pixels in the image. The summation over i runs from 1 to N, covering every pixel in the input image.

**Is colour a good feature choice?**

Colour is a reasonable but limited feature choice for this scene. It works well for semantic segmentation (separating chairs from table from background) if the objects have distinct colours. However, colour alone is problematic when:
- Different objects share similar colours (e.g., wooden chairs and a wooden table would have the same brown/wood colour).
- The same object has multiple colours due to texture or lighting variations.
- Shadows create colour variations within a single object.

For the given scene with chairs and table, colour may partially segment the scene but would likely merge objects of similar colour into one cluster and split objects with colour variation into multiple clusters.

#### (ii) K-Means with Pixel Coordinates for Instance Segmentation [4 marks]

If the feature vector x_i consists solely of the pixel coordinates (x, y) of pixel i, then k-means would cluster pixels based purely on their spatial proximity in the image. Each cluster would correspond to a spatially contiguous region (approximately a Voronoi cell around each cluster centre).

**Performance for instance segmentation:**

This approach would perform poorly for instance segmentation of chairs/table because:

1. **No appearance information**: The algorithm has no knowledge of what a pixel looks like (colour, texture), only where it is. It cannot distinguish between a chair pixel and a nearby background pixel.

2. **Spatial grouping only**: K-means with coordinates would simply partition the image into k roughly equally-sized spatial regions (like a spatial tiling), regardless of object boundaries.

3. **Objects spanning multiple regions**: A large object (e.g., the table) might be split across multiple clusters if it spans a large spatial extent.

4. **Multiple objects in one region**: Two nearby chairs might be grouped into the same cluster if they are spatially close.

5. **Shape ignorance**: Instance segmentation requires understanding that objects have specific shapes. Coordinate-based clustering produces convex (approximately circular/elliptical) regions that do not conform to object contours.

In summary, pixel coordinates alone provide no discriminative power for distinguishing between different object instances. A better approach would combine spatial coordinates with appearance features (colour, texture) to leverage both where and what information.

#### (iii) Neighbourhood Term f(x_i, x_j) [3 marks]

Adding a neighbourhood term f(x_i, x_j) to the objective function introduces a spatial regularisation constraint that encourages neighbouring pixels to be assigned to the same cluster.

**Effects:**

1. **Spatial coherence**: The modified objective penalises solutions where adjacent pixels are assigned to different clusters. This produces smoother, more spatially coherent segmentations rather than noisy, fragmented results.

2. **Noise reduction**: Isolated pixels that might be assigned to the wrong cluster based solely on their feature value are now "pulled" toward the label of their neighbours, effectively reducing salt-and-pepper noise in the segmentation map.

3. **Edge preservation**: Depending on the form of f(x_i, x_j), the term can be designed to allow discontinuities where there are strong feature differences between neighbours (like an edge-aware smoothness prior). For example, f could be small when ||x_i - x_j|| is large (indicating a genuine boundary) and large when features are similar (encouraging same-label assignment).

4. **MRF connection**: This transforms k-means into something resembling a Markov Random Field (MRF) energy minimisation, where the unary term is the distance to cluster centres and the pairwise term encodes neighbourhood relationships.

The overall effect is a segmentation that better respects object boundaries while avoiding over-segmentation of textured regions.

### (b) Pictorial Structure for Instance Segmentation

#### (i) Measurable Features of a Part [2 marks]

For each part V_i of the pictorial structure (e.g., a chair leg, seat, or backrest), measurable features include:

- **Position**: The (x, y) location of the part in the image.
- **Orientation**: The angle/direction of the part (e.g., a chair leg is roughly vertical).
- **Scale**: The size of the part in the image (which varies with distance from camera).
- **Appearance**: Colour histogram, texture descriptor, or edge template that characterises what the part looks like.
- **Aspect ratio**: The width-to-height ratio of the bounding region for the part.

These features collectively define the configuration of a part and allow the model to evaluate whether a candidate detection at a particular location, scale, and orientation matches the expected appearance of that part.

#### (ii) Cost Function Components [2 marks]

L = Sigma_i C(V_i) + Sigma_(i,j) in E C(V_i, V_j)

- **C(V_i) - Unary cost (appearance cost)**: This measures how well the image evidence at the proposed location/configuration of part V_i matches the expected appearance model for that part. A low cost means the local image patch looks like the part should (e.g., the edge pattern matches a chair leg template). It penalises placing a part where the image evidence does not support its presence.

- **C(V_i, V_j) - Pairwise cost (deformation/geometric cost)**: This measures the geometric compatibility between connected parts V_i and V_j. It penalises configurations where the relative position, orientation, or scale between two connected parts deviates from the expected spatial relationship. For example, if V_i is the chair seat and V_j is a chair leg, this term penalises configurations where the leg is too far from or at an implausible angle relative to the seat.

#### (iii) Search Space for Fitting the Pictorial Model [3 marks]

The search space for fitting the pictorial model is the space of all possible configurations of all 6 parts simultaneously. For each part V_i, we must search over:

- Position: (x, y) - if the image is W x H pixels, there are W x H possible positions.
- Orientation: theta - discretised into O possible angles.
- Scale: s - discretised into S possible scales.

For a single part, the configuration space has size W x H x O x S.

For 6 parts, the full joint search space is (W x H x O x S)^6, which is exponentially large.

However, because the pictorial structure is a tree-structured graph (not fully connected), dynamic programming can be applied. The tree structure allows efficient inference by passing messages from leaf nodes to the root, reducing the complexity from exponential to linear in the number of parts. Specifically, with distance transform tricks, the complexity becomes O(6 x W x H x O x S) rather than the full combinatorial explosion.

Without the tree structure, exact inference would require exhaustive search over all joint configurations, which is computationally intractable.

#### (iv) Challenges of Pictorial Structure for Detecting Individual Chairs [3 marks]

1. **Multiple instances**: The pictorial structure is typically designed to detect a single instance of an object. When multiple chairs are present in the scene, the model must be run multiple times or extended to handle multiple detections. Parts from different chair instances may be incorrectly connected (e.g., a leg from chair A matched with the seat of chair B).

2. **Occlusion**: In a scene with chairs around a table, chairs partially occlude each other. Some parts may be completely hidden, making their appearance cost unreliable. The model must handle missing or occluded parts gracefully, which standard pictorial structures do not inherently support.

3. **Viewpoint variation**: Different chairs in the scene may be viewed from different angles (front, side, back). The geometric relationships between parts change significantly with viewpoint, meaning a single set of pairwise constraints may not capture all valid configurations.

4. **Self-similarity of parts**: Chair legs from different chairs look identical. The algorithm may find it difficult to determine which legs belong to which chair, especially when chairs are close together. This part-to-instance association problem is a fundamental challenge.

5. **Deformable geometry**: Real chairs vary in design, and the spatial relationships between parts may differ across chair types (e.g., stacking chairs vs. armchairs), making a single rigid model insufficient.

## Question 2: Stereo Vision (20 marks)

### (a) Epipolar Geometry and Disparity Gradient

#### (i) Disparity Gradient and Its Constraint [3 marks]

**Disparity Gradient (DG):** The disparity gradient between two matched point pairs is defined as the ratio of the difference in disparities to the mean separation of the points. For two points P1 and P2 with disparities d1 and d2, the disparity gradient is:

DG = |d1 - d2| / (separation between points in cyclopean view)

More formally, if (l1, r1) and (l2, r2) are the left and right image positions of two corresponding points:

DG = |(l1 - r1) - (l2 - r2)| / (0.5 * |(l1 + r1) - (l2 + r2)|)

**Disparity Gradient Constraint:** The disparity gradient limit states that for valid matches, the disparity gradient should be less than a threshold (typically DG < 1, with DG < 2 as an absolute physical limit). When DG = 1, one of the viewing rays is parallel to the surface. When DG >= 2, the surface would need to fold back on itself, which is physically impossible for opaque surfaces.

**Constraint on viewed surfaces:** The DG constraint transforms into a constraint on the orientation of surfaces in the scene. Specifically, DG < 1 implies that the surface being viewed must be facing toward both cameras (i.e., the surface normal has a component pointing toward both cameras). Surfaces with DG approaching 2 would be nearly edge-on to one camera (a fronto-parallel surface has DG = 0). This means the constraint effectively limits the maximum slant of surfaces that can be reliably matched, eliminating physically implausible or highly oblique surface configurations.

#### (ii) Epipolar Geometry Constraining Search [3 marks]

Epipolar geometry provides a fundamental constraint that reduces the stereo correspondence search from 2D to 1D.

**Explanation:** Given a point p in the left image, its corresponding point in the right image must lie on a specific line called the epipolar line. This arises because:

1. The point p, the left camera centre O_L, and the 3D point P are coplanar with the right camera centre O_R (forming the epipolar plane).
2. The intersection of this epipolar plane with the right image plane defines the epipolar line.

**Sketch description:** Consider two cameras (O_L and O_R) observing a 3D point P. The baseline connects O_L and O_R. The epipolar plane is defined by P, O_L, and O_R. The epipolar line e_R in the right image is where this plane intersects the right image plane. Similarly, e_L in the left image is where the plane meets the left image plane. The epipoles (e_l, e_r) are where the baseline intersects each image plane.

**Constraint:** Instead of searching the entire 2D right image for a match to a left image point, we only need to search along the 1D epipolar line. For rectified stereo pairs (where cameras have parallel optical axes), all epipolar lines are horizontal, so corresponding points lie on the same scanline row. This reduces computational complexity from O(N^2) per pixel to O(N).

#### (iii) Zhang and Shan's Progressive Scheme [4 marks]

Zhang and Shan's progressive scheme combines the epipolar constraint and the disparity gradient constraint in a coarse-to-fine manner to progressively build up a reliable disparity map:

**Step 1 - Seed matches along epipolar lines:** First, establish a sparse set of highly reliable "seed" correspondences using the epipolar constraint. These are matches with very high confidence (e.g., using cross-correlation with strict thresholds on distinctive features). The epipolar constraint limits search to 1D scanlines.

**Step 2 - Local propagation using DG constraint:** From each seed match, propagate to neighbouring points. For a seed match with known disparity d, nearby points should have similar disparities (smooth surface assumption). The DG constraint limits how much the disparity can change between neighbouring matches - only candidate matches satisfying DG < threshold (typically 1) relative to existing confirmed matches are accepted.

**Step 3 - Progressive expansion:** Newly confirmed matches become seeds for further propagation. The process iterates, gradually expanding the matched region outward from initial seeds. At each step, the DG constraint ensures physical plausibility of new matches relative to already-established neighbours.

**Step 4 - Multi-resolution/iterative refinement:** The scheme may operate at multiple resolutions, establishing matches at coarse scales first (where large features are easy to match) and refining at finer scales. Ambiguous regions are revisited as more context from surrounding matches becomes available.

**Advantages:** This progressive approach avoids the combinatorial explosion of trying all possible matches simultaneously. It leverages the DG constraint to prune the search space locally, while the epipolar constraint provides global 1D restriction. The result is a dense disparity map built incrementally from high-confidence sparse matches.

### (b) Scanline Stereo Method

#### (i) Possible Assignments to a Pixel [3 marks]

In scanline stereo, each pixel along a scanline in the left image can be assigned one of the following:

1. **Match**: The pixel is matched to a specific pixel in the right scanline. This means both pixels are projections of the same 3D surface point visible to both cameras. A match (l_i, r_j) indicates left pixel i corresponds to right pixel j, with disparity d = i - j.

2. **Left occlusion (left pixel unmatched)**: The left pixel has no corresponding pixel in the right image. This means the 3D point visible in the left camera is occluded (hidden) in the right camera's view - something blocks it from the right camera's perspective.

3. **Right occlusion (right pixel unmatched)**: A right pixel has no corresponding pixel in the left image. This means the 3D point visible in the right camera is occluded in the left camera's view.

Each assignment carries a cost: matches are penalised by dissimilarity between the pixel values (e.g., absolute intensity difference), while occlusions carry a fixed penalty cost reflecting how unlikely occlusions are relative to matches.

#### (ii) Ordering Constraint [3 marks]

**The ordering constraint** states that the left-to-right ordering of matched points along an epipolar line (scanline) must be preserved in both images. If point A is to the left of point B in the left image, then A's match must be to the left of B's match in the right image.

Formally: if (l_i, r_a) and (l_j, r_b) are two matches with i < j, then the ordering constraint requires a < b.

**Violation example:** The ordering constraint is violated by thin objects or narrow protruding surfaces in front of a background. Consider a thin vertical pole viewed by two cameras. In the left image, the background pixel on the left side of the pole is visible, and in the right image, the background pixel on the right side of the pole is visible. The relative left-right ordering of these background points reverses between the two views because the pole's position shifts differently in each image. Specifically, a background point that appears to the left of the pole in the left image may appear to the right of the pole in the right image due to parallax, violating the ordering constraint.

Transparent or reflective surfaces can also violate this constraint, but the classic case is a thin foreground occluder.

## Question 3: Optical Flow and Visual Tracking (20 marks)

### (a) Horn-Schunck and Lucas-Kanade

#### (i) Global vs. Local Methods [2 marks]

**Horn-Schunck is global** because it formulates optical flow estimation as a single energy minimisation over the entire image simultaneously. The objective function includes a smoothness term that couples the flow at every pixel to its neighbours, meaning the flow at any one pixel depends (transitively) on all other pixels in the image. The solution is obtained by solving a system of equations involving all pixels at once (typically via iterative methods like Gauss-Seidel).

**Lucas-Kanade is local** because it estimates optical flow independently at each pixel using only information from a small local neighbourhood (window) around that pixel. The flow (u, v) at a pixel is computed by solving an overdetermined system of brightness constancy equations from the surrounding patch (e.g., 5x5 or 7x7 window). The computation at one pixel does not depend on flow estimates at distant pixels.

#### (ii) Lucas-Kanade with Image Pyramid for Large Motion [5 marks]

The standard Lucas-Kanade method assumes small displacements (linearisation of brightness constancy requires motion smaller than ~1 pixel). For large motions, this assumption is violated. The image pyramid (coarse-to-fine) approach overcomes this limitation:

**Image Pyramid Construction:**
1. Build a Gaussian pyramid for both frames by repeatedly smoothing and subsampling. Level 0 is the original resolution, Level 1 is half-resolution, Level 2 is quarter-resolution, etc.
2. At coarser levels, large motions in the original image appear as small motions (scaled down by the subsampling factor).

**Coarse-to-Fine Estimation:**
1. **Start at coarsest level (Level L):** At the lowest resolution, even large motions appear small (a 16-pixel motion at Level 0 becomes a 1-pixel motion at Level 4). Apply standard Lucas-Kanade to estimate the flow (u_L, v_L) at this coarse level.

2. **Propagate to next finer level:** Upsample the coarse flow estimate by factor 2 (and multiply values by 2 to account for scale change). Use this upsampled flow as an initial estimate at Level L-1.

3. **Warp and compute residual:** Warp the second frame at Level L-1 using the initial flow estimate. Now compute Lucas-Kanade on the warped image to find the residual (remaining small) motion.

4. **Accumulate:** Add the residual flow to the initial estimate to get the refined flow at Level L-1.

5. **Repeat:** Continue propagating to finer levels until reaching Level 0 (original resolution).

**Why this works:** At each level, Lucas-Kanade only needs to estimate a small residual motion (within its linear approximation validity), while the large component has already been captured at coarser scales. The final result can handle motions of many pixels while maintaining the accuracy of Lucas-Kanade.

#### (iii) Aperture Problem Analysis [3 marks]

Consider an opaque square moving to the top-right:

**(a) Aperture on the bottom edge:**
The bottom edge is a horizontal line. Through the aperture, we observe this edge moving upward, but we cannot determine the horizontal component of motion. The edge provides constraint only in the direction perpendicular to it (vertical direction). The horizontal motion along the edge is ambiguous.

**Result: Motion CANNOT be fully recovered.** Only the normal component (upward) is measurable. This is the classic aperture problem - a straight edge only constrains motion perpendicular to itself.

**(b) Aperture on the interior (uniform region):**
In the interior of a uniformly coloured square, there is no spatial gradient - the intensity is constant. Without any image gradient, the brightness constancy equation I_x*u + I_y*v + I_t = 0 gives 0*u + 0*v + 0 = 0 (assuming uniform intensity over time within the region), providing no constraint at all.

**Result: Motion CANNOT be recovered.** There is no texture or structure to track. Both components of motion are ambiguous.

**(c) Aperture on the right edge with a circle (corner/feature):**
The circle on the right edge creates a distinctive feature - effectively a corner or junction where multiple gradient directions exist. With gradients in at least two independent directions within the aperture, the system of equations is well-conditioned (the structure tensor has two significant eigenvalues).

**Result: Motion CAN be fully recovered.** The circle provides texture that constrains motion in both directions. The presence of multiple gradient orientations (from the circle's boundary and the edge) resolves the aperture problem.

### (b) CONDENSATION Particle Filter

#### (i) What Particles Represent [3 marks]

In the CONDENSATION (Conditional Density Propagation) algorithm, each particle represents a hypothesised state of the tracked object at the current time step. Specifically:

- Each particle is a sample from the state space, encoding a possible configuration of the target. The state typically includes position (x, y), scale, velocity, and possibly shape parameters of the object being tracked.
- Each particle has an associated weight representing how likely that hypothesis is given the current observation (image evidence).
- The collection of all weighted particles approximates the posterior probability distribution P(x_t | Z_t) over the object state given all observations up to time t.

**Single vs. multiple targets:** The standard CONDENSATION algorithm tracks a **single target**. However, because it maintains a full distribution (not just a single point estimate), the particle set can represent multi-modal distributions. This means it can maintain multiple hypotheses about where a single object might be (e.g., during ambiguous situations), but it is not designed for multi-target tracking. For multiple targets, extensions such as separate particle filters per target or joint state representations would be needed.

#### (ii) Meaning of P(z_{t+1} | x_{t+1}) [2 marks]

**P(z_{t+1} | x_{t+1})** is the **observation likelihood** (also called the measurement model or observation model). It represents the probability of observing measurement z_{t+1} (the current image evidence) given that the object is in state x_{t+1}.

In practice, this measures how well the image evidence at the hypothesised location/configuration matches the appearance model of the target. For example, if tracking based on colour histograms, P(z_{t+1} | x_{t+1}) would be high when the colour histogram of the image region defined by state x_{t+1} closely matches the target's reference histogram.

**How it is used in CONDENSATION:** After predicting particles forward in time (using the dynamics model) and observing the new frame, each particle's weight is set proportional to P(z_{t+1} | x_{t+1}). Particles that explain the observation well receive high weights, while those in unlikely configurations receive low weights. These weights are then used in the resampling step, where particles with higher weights are more likely to be selected for the next generation, focusing computational resources on promising regions of state space.

#### (iii) Approximation of the Prediction Equation [5 marks]

The prediction equation is:

P(x_{t+1} | Z_t) = integral P(x_{t+1} | x_t) P(x_t | Z_t) dx_t

This integral says: to predict where the object will be at t+1 given past observations Z_t, we must consider all possible current states x_t (weighted by their posterior probability) and propagate each forward through the dynamics model P(x_{t+1} | x_t).

**Particle Approximation:**

1. **Represent the posterior with particles:** The posterior P(x_t | Z_t) is approximated by a set of N weighted particles {(x_t^(n), w_t^(n))}_{n=1}^{N}:

   P(x_t | Z_t) approximately equals Sigma_{n=1}^{N} w_t^(n) * delta(x_t - x_t^(n))

   where delta is the Dirac delta function and weights are normalised to sum to 1.

2. **Resample:** Draw N particles from the current set with probability proportional to their weights. This gives an unweighted set of particles {x_t^(n)} that are distributed according to P(x_t | Z_t). Particles with high weights may be selected multiple times; those with low weights may not be selected at all.

3. **Propagate through dynamics:** For each resampled particle x_t^(n), draw a new sample from the dynamics model:

   x_{t+1}^(n) ~ P(x_{t+1} | x_t^(n))

   This typically involves applying a deterministic motion model plus random noise: x_{t+1}^(n) = A * x_t^(n) + B * w_n, where w_n is a random noise sample.

4. **Result:** The collection of propagated particles {x_{t+1}^(n)} approximates the predictive distribution P(x_{t+1} | Z_t).

**Why this is plausible:**

- By the law of large numbers, as N approaches infinity, the empirical distribution of particles converges to the true distribution. The Monte Carlo approximation replaces an intractable integral over continuous state space with a finite sum over samples.
- The resampling step ensures particles concentrate in high-probability regions, making efficient use of finite samples.
- The stochastic propagation step correctly implements marginalisation: by sampling from P(x_{t+1} | x_t) for many different x_t values drawn from P(x_t | Z_t), we are performing Monte Carlo integration of the prediction integral.
- The approach makes no assumptions about the form of the distributions (unlike Kalman filters which require Gaussianity), allowing it to represent multi-modal, non-Gaussian posteriors - critical for robust tracking under ambiguity.

## Question 4: Convolutional Neural Networks (20 marks)

### (a) CNN Architecture Analysis

#### (i) Why Filtering is 3D and Pooling is 2D [2 marks]

**Filtering is 3D** because convolutional filters operate across the full depth (number of channels) of the input volume. A filter has dimensions height x width x depth, where depth equals the number of input channels. For example, a 5x5 filter applied to an RGB image is actually 5x5x3, computing a dot product across all three colour channels at each spatial position. In deeper layers, a filter spans all feature maps from the previous layer. The "3D" nature refers to the filter extending through the channel dimension.

**Pooling is 2D** because pooling (e.g., max-pooling or average-pooling) operates independently on each feature map (channel). It reduces the spatial dimensions (height and width) by summarising local patches within each channel separately, but does not combine information across channels. A 2x2 max-pool with stride 2 is applied to each feature map independently, halving the spatial dimensions while keeping the number of channels unchanged.

#### (ii) Number of Parameters Calculation [4 marks]

**Input:** 256 x 256 x 3 (RGB image)

**First convolutional layer:** 64 filters of size 5 x 5
- Each filter has dimensions: 5 x 5 x 3 (spatial size x input channels)
- Parameters per filter: 5 x 5 x 3 = 75
- Total parameters for layer 1: 75 x 64 = **4,800**

**Second convolutional layer:** 64 filters of size 3 x 3
- Input to this layer has 64 channels (from the 64 feature maps of layer 1)
- Each filter has dimensions: 3 x 3 x 64
- Parameters per filter: 3 x 3 x 64 = 576
- Total parameters for layer 2: 576 x 64 = **36,864**

**Total convolutional parameters (ignoring bias):** 4,800 + 36,864 = **41,664**

Note: If the fully-connected classifier layer is included, its parameters would depend on the spatial dimensions after pooling. For example, if two 2x2 pooling layers reduce 256x256 to 64x64, then the FC layer input would be 64 x 64 x 64 = 262,144 neurons, and connecting to (say) 1000 output classes would add 262,144,000 parameters - dwarfing the convolutional parameters.

**Why large labelled datasets are needed:**

With tens of thousands (or millions when including FC layers) of learnable parameters, the network has enormous capacity to memorise training data. Without sufficient labelled training examples:
- **Overfitting**: The network will memorise the training set rather than learning generalisable features, performing poorly on unseen data.
- **Poor generalisation**: With too few examples relative to parameters, many valid parameter settings fit the training data equally well, but most will not generalise.
- **Statistical learning theory**: The VC dimension (capacity) of deep networks is very high, requiring proportionally more training samples to achieve good generalisation bounds.

Large labelled datasets (e.g., ImageNet with 1.2 million images) provide sufficient diversity and volume to regularise learning and enable the network to discover robust, transferable visual features.

#### (iii) Training the CNN with Fully-Connected Classifier [4 marks]

The CNN with fully-connected classifier is trained end-to-end using backpropagation and stochastic gradient descent (SGD):

**Forward pass:**
1. Input image passes through convolutional layers (convolution + activation function like ReLU + pooling).
2. The output feature maps are flattened into a 1D vector.
3. This vector passes through one or more fully-connected layers.
4. The final layer produces class scores (logits), typically passed through softmax to produce a probability distribution over classes.

**Loss computation:**
- Compare the predicted probabilities to the ground-truth label using a loss function, typically cross-entropy loss: L = -Sigma_c y_c * log(p_c), where y_c is the one-hot ground truth and p_c is the predicted probability for class c.

**Backward pass (Backpropagation):**
1. Compute the gradient of the loss with respect to the output layer weights.
2. Propagate gradients backward through the fully-connected layers using the chain rule.
3. Continue propagating through the flattening operation and into the convolutional layers.
4. For convolutional layers, gradients are computed with respect to filter weights by convolving the incoming gradient with the input feature map.
5. Gradients through pooling layers are routed back to the locations that contributed to the pooled output (e.g., for max-pooling, gradient goes only to the location of the maximum value).

**Weight update:**
- Update all parameters (convolutional filters and FC weights) using the computed gradients: w := w - eta * dL/dw, where eta is the learning rate.

**Training procedure:**
1. Initialise weights (e.g., Xavier/He initialisation).
2. For each epoch, shuffle training data and form mini-batches.
3. For each mini-batch: forward pass, compute loss, backward pass, update weights.
4. Monitor validation loss to prevent overfitting; apply regularisation (dropout, weight decay, data augmentation).
5. Continue until convergence or early stopping criterion is met.

### (b) CNN Advances

#### (i) GoogLeNet Inception Unit [3 marks]

**The problem:** Standard convolutional layers apply filters of a single fixed size (e.g., all 3x3 or all 5x5). The optimal filter size depends on the local structure in the image - small filters capture fine details while large filters capture broader context. Choosing a single size forces a compromise.

**How Inception overcomes this:** The Inception unit applies multiple filter sizes in parallel at the same layer:
- 1x1 convolutions (capture channel-wise patterns)
- 3x3 convolutions (capture local spatial patterns)
- 5x5 convolutions (capture larger spatial patterns)
- 3x3 max-pooling (capture dominant features)

All branches receive the same input, and their outputs are concatenated along the channel dimension to form the layer's output.

**Key design choice:** To manage computational cost, 1x1 convolutions are used as dimensionality reduction bottlenecks before the expensive 3x3 and 5x5 convolutions. This reduces the number of input channels before applying larger filters.

**Advantage:** The network does not need to "decide" which filter size is appropriate - it learns to combine multi-scale features automatically. This allows the network to capture patterns at multiple spatial scales simultaneously, improving representational power without requiring manual architecture decisions about filter size. It also enables deeper networks with fewer parameters compared to stacking large filters.

#### (ii) ResNet Skip Connections and Vanishing Gradient [3 marks]

**The vanishing gradient problem:** In very deep networks, gradients computed during backpropagation must pass through many layers. At each layer, the gradient is multiplied by the layer's weights and passed through activation function derivatives. After many multiplications, gradients can become exponentially small (vanish), meaning early layers receive negligible gradient updates and cannot learn effectively.

**How skip connections help:** ResNet introduces residual blocks where the output is:

H(x) = F(x) + x

where F(x) is the transformation learned by the stacked layers (convolutions, batch norm, ReLU), and x is the input passed directly via the skip connection.

**Gradient flow analysis:** During backpropagation, the gradient with respect to the input x is:

dL/dx = dL/dH * dH/dx = dL/dH * (dF/dx + I)

The identity term (I) ensures that the gradient has a direct path back through the network without being diminished by weight multiplications. Even if dF/dx is very small (vanishing), the gradient dL/dH * I = dL/dH passes through unchanged. This creates a "gradient highway" that allows gradients to flow directly from later layers to earlier layers.

**Practical benefit:** This enables training of extremely deep networks (100+ layers). The network only needs to learn the residual F(x) = H(x) - x, which is often easier to optimise (if the identity mapping is near-optimal, the network just needs to push F(x) toward zero, which is simpler than learning a complete transformation from scratch).

#### (iii) Transfer Learning from VGG for Image Segmentation [4 marks]

VGG trained on ImageNet (1000-class object classification) can be repurposed for image segmentation through transfer learning:

**Step 1 - Feature extractor (encoder):**
Take the pre-trained VGG network and remove the final fully-connected classification layers. The remaining convolutional layers serve as a powerful feature extractor that has learned hierarchical visual representations (edges, textures, parts, objects) from ImageNet's diverse images. These features are largely task-agnostic and transfer well to new vision tasks.

**Step 2 - Add segmentation decoder:**
Append new layers that produce a dense, pixel-wise output (same spatial dimensions as input):
- **Upsampling layers**: Use transposed convolutions (deconvolutions) or bilinear upsampling followed by convolutions to progressively increase spatial resolution from the compressed feature maps back to full image resolution.
- **Skip connections from encoder**: Connect feature maps from earlier VGG layers (which retain fine spatial detail) to the decoder via concatenation or addition. This helps recover precise boundary information lost during pooling (as in U-Net/FCN architectures).
- **Final 1x1 convolution**: Map the decoded features to per-pixel class predictions (C channels for C segmentation classes, followed by softmax).

**Step 3 - Fine-tuning strategy:**
- **Freeze early VGG layers**: Low-level features (edges, colours, textures) are highly transferable - keep these fixed initially.
- **Train decoder from scratch**: The new upsampling/segmentation layers have random weights and need full training.
- **Optionally fine-tune later VGG layers**: With a small learning rate, adapt the higher-level features to the segmentation task's specific domain.
- **Train on segmentation dataset**: Use pixel-wise cross-entropy loss comparing predicted segmentation maps to ground-truth masks.

**Why this works:** ImageNet pre-training gives VGG rich visual features that generalise across tasks. The segmentation task benefits from these features without needing millions of densely-labelled segmentation images (which are expensive to annotate). Transfer learning allows achieving good segmentation performance with relatively small task-specific datasets.

This approach is exemplified by Fully Convolutional Networks (FCN), SegNet, and U-Net architectures, all of which use pre-trained classification networks as encoders for dense prediction tasks.
