## Question 1: Segmentation (25 marks)

### (a) K-Means Clustering [7 marks]

**Given:** Data set {2, 4, 6, 8, 10, 12}, k = 2, initial means μ₁ = 2, μ₂ = 4.

**Algorithm:** Assign each point to the nearest mean, then recompute means as the average of assigned points.

#### Iteration 1

| Point | Distance to μ₁=2 | Distance to μ₂=4 | Assigned Cluster |
|-------|-------------------|-------------------|------------------|
| 2     | 0                 | 2                 | C1               |

| 6     | 4                 | 2                 | C2               |
| 8     | 6                 | 4                 | C2               |
| 10    | 8                 | 6                 | C2               |
| 12    | 10                | 8                 | C2               |

- C1 = {2} → μ₁ = 2/1 = **2**
- C2 = {4, 6, 8, 10, 12} → μ₂ = 40/5 = **8**

| Point | Distance to μ₁=2 | Distance to μ₂=8 | Assigned Cluster |

| 6     | 4                 | 2                 | C2               |

| 12    | 10                | 4                 | C2               |

- C1 = {2, 4} → μ₁ = 6/2 = **3**
- C2 = {6, 8, 10, 12} → μ₂ = 36/4 = **9**

#### Iteration 3

### (b) Four Problems with K-Means Clustering [4 marks]

1. **Sensitivity to initialisation:** The algorithm can converge to different local minima depending on the initial placement of cluster centres, potentially yielding suboptimal solutions.

2. **Need to specify k in advance:** The number of clusters must be chosen before running the algorithm, but the true number of clusters in the data is often unknown.

3. **Sensitivity to outliers:** Outliers can significantly distort cluster means since k-Means minimises squared Euclidean distance, pulling centroids away from the true cluster centre.

4. **Assumes spherical clusters of equal size:** K-Means uses Euclidean distance and assigns each point to the nearest mean, which implicitly assumes clusters are spherical (isotropic) and roughly equal in size. It fails on elongated, irregularly shaped, or differently sized clusters.

---

### (c) Solutions and Extensions [6 marks]

1. **K-Means++ initialisation (addresses problem 1):** Instead of random initialisation, choose the first centre uniformly at random, then select subsequent centres with probability proportional to the squared distance from the nearest existing centre. This spreads initial centres and leads to better convergence.

2. **Elbow method or silhouette analysis (addresses problem 2):** Run k-Means for a range of k values and plot the within-cluster sum of squares (WCSS). The "elbow" in the curve suggests an appropriate k. Alternatively, compute silhouette scores to evaluate cluster quality at each k.

3. **K-Medoids / Median-based approaches (addresses problem 3):** Use the actual data point closest to the cluster centre (medoid) rather than the mean. This makes the algorithm more robust to outliers since the medoid is less influenced by extreme values.

4. **Gaussian Mixture Models / EM algorithm (addresses problem 4):** Instead of hard assignment, model each cluster as a Gaussian distribution with its own covariance matrix. The Expectation-Maximisation algorithm estimates parameters, allowing clusters of different shapes, sizes, and orientations. Points receive soft probabilistic assignments to clusters.

### (d) Two Region-Based Segmentation Methods [8 marks]

#### Method 1: Region Growing

Region growing starts with seed points and iteratively expands regions by incorporating neighbouring pixels that satisfy a homogeneity criterion.

**Algorithm:**

1. Select one or more seed pixels (manually or automatically based on some criterion such as intensity extrema).
2. Define a homogeneity predicate (e.g., pixel intensity within a threshold of the region mean).
3. Examine the 4-connected or 8-connected neighbours of each seed pixel.
4. If a neighbouring pixel satisfies the homogeneity predicate, add it to the region and update the region statistics (mean, variance).
5. Repeat step 3-4 for all newly added pixels.
6. Stop when no more neighbouring pixels satisfy the predicate.
7. Repeat for remaining unseeded pixels or other seed points.

**Characteristics:** Produces connected regions, is sensitive to seed selection and threshold choice, and can be slow for large images.

#### Method 2: Split and Merge (Quad-tree based)

Split and merge recursively divides the image into quadrants and then merges adjacent regions that are sufficiently similar.

**Algorithm:**

1. Start with the entire image as a single region.
2. **Split phase:** For each region, test the homogeneity predicate (e.g., variance of pixel intensities below a threshold).
   - If the region is NOT homogeneous, split it into four equal quadrants.
   - Repeat recursively until all regions are homogeneous or reach a minimum size.
3. **Merge phase:** For each pair of adjacent regions (not necessarily from the same parent):
   - If merging the two regions still satisfies the homogeneity predicate, merge them into one region.
   - Update region statistics after merging.
4. Repeat the merge step until no more merges are possible.
5. Output the final set of regions as the segmentation.

**Characteristics:** The quad-tree structure provides a multi-resolution approach. The split phase ensures no region contains highly disparate pixels; the merge phase corrects over-segmentation caused by the rigid grid splitting.

## Question 2: Binocular Stereo (25 marks)

### (a) Disparity and Depth Computation [4 marks]

**Disparity** is the difference in horizontal position of a point's projection in the left and right images of a stereo pair. If a 3D point P projects to position xL in the left image and xR in the right image, the disparity is:

d = xL - xR

Disparity is inversely proportional to depth. For a parallel stereo camera setup with focal length f and baseline b (distance between camera centres), the relationship is:

Z = (f × b) / d

Points closer to the cameras have larger disparity (they shift more between views), while distant points have smaller disparity (approaching zero at infinity). By computing disparity for each pixel and applying this formula, we can reconstruct a depth map of the scene.

---

### (b) Camera Setups [5 marks]

Three characteristics by which stereo camera setups may differ:

1. **Baseline distance:** The separation between the two cameras. A larger baseline provides better depth resolution for distant objects but creates larger occluded regions and makes correspondence harder. A smaller baseline is better for nearby objects but gives less precise depth estimates for far objects.

2. **Vergence/orientation:** Cameras can be arranged in a parallel configuration (optical axes are parallel) or a converging (verged) configuration where optical axes intersect at a fixation point. Parallel setups simplify the geometry; verged setups can increase the overlapping field of view.

3. **Focal length and sensor resolution:** Cameras may have different lens focal lengths or sensor resolutions, which affect the field of view, magnification, and the scale of image features.

**Why calibration is necessary:** Calibration determines the intrinsic parameters (focal length, principal point, lens distortion) and extrinsic parameters (relative position and orientation of the two cameras). Without calibration, we cannot accurately rectify the images (to make epipolar lines horizontal), compute the correct relationship between disparity and depth, or account for lens distortion. Calibration transforms pixel coordinates into consistent geometric measurements needed for triangulation.

---

### (c) Monocular Cues and Algorithm Design [5 marks]

Humans do **not** strictly require high-level monocular cues (such as perspective, familiar size, occlusion, or texture gradients) for binocular stereo perception. Psychophysical experiments with random dot stereograms (Julesz, 1960s) demonstrated that humans can perceive depth from stereo disparity alone, even when individual images contain no recognisable objects or monocular depth cues. The 3D structure is perceived solely through binocular fusion of the dot patterns.

**Influence on algorithm design:** This finding suggests that stereo algorithms do not need to perform high-level scene understanding or object recognition before computing depth. Instead, algorithms can work at the pixel or local patch level, matching image regions based purely on intensity/texture patterns. This justifies:

- Area-based correlation methods that match small rectangular windows between images
- The use of purely local features (pixel intensities, gradients) for correspondence
- Processing stereo matching as a low-level vision task independent of object recognition

However, incorporating smoothness constraints and global optimisation (like graph cuts or dynamic programming) can improve results, acting as implicit assumptions about scene structure without requiring explicit object-level reasoning.

---

### (d) The Correspondence Problem [7 marks]

**The correspondence problem** is the task of determining which point in the left image corresponds to the same 3D scene point in the right image. For every pixel in the left image, we must find its matching pixel in the right image to compute disparity. This is challenging because:

- Multiple pixels may have similar intensities/textures, creating ambiguity
- Some points visible in one image may be occluded in the other
- Reflective surfaces, transparent objects, and repetitive patterns cause false matches
- Foreshortening means the same surface patch may look different in each view

**How camera geometry simplifies it:**

The **epipolar constraint** dramatically reduces the search space. For a calibrated stereo pair, the geometry dictates that the corresponding point in the right image must lie along a specific line called the **epipolar line**. This reduces the 2D search to a 1D search.

For a **rectified** stereo pair (achieved through calibration and image warping):
1. Epipolar lines become horizontal scanlines
2. The corresponding point for pixel (x, y) in the left image lies on the same row y in the right image
3. Search is restricted to a 1D range along that scanline
4. Disparity is simply the horizontal shift: d = xL - xR

Additionally, the **ordering constraint** (for opaque, continuous surfaces) states that if point A is to the left of point B in one image, the same ordering holds in the other image. This allows dynamic programming approaches along scanlines.

---

### (e) Dense vs Sparse Disparity Maps [4 marks]

**Dense disparity maps** compute a disparity value for every pixel in the image.

- **Conditions favouring dense maps:** Well-textured scenes, applications requiring complete 3D reconstruction (e.g., autonomous navigation, 3D modelling), scenes without large uniform regions.
- **Pros:** Complete depth information; directly yields a full 3D surface reconstruction.
- **Cons:** Computationally expensive; unreliable in textureless regions where matching is ambiguous; may produce errors that require post-processing (hole-filling, smoothing).

**Sparse disparity maps** compute disparity only at selected feature points (corners, edges, high-texture regions).

- **Conditions favouring sparse maps:** Scenes with distinct features, applications where only partial depth information is needed (e.g., obstacle detection, object localisation), real-time systems with limited computation.
- **Pros:** Faster computation; matches are more reliable since features are distinctive; fewer false matches.
- **Cons:** Incomplete depth information; interpolation needed between sparse points may be inaccurate; misses information in textureless regions entirely.

## Question 3: Motion Analysis and Optic Flow (25 marks)

### (a) Importance of Motion Analysis [4 marks]

1. **Surveillance and security:** Detecting intruders, abandoned objects, or abnormal behaviour in monitored environments (airports, car parks, public spaces).

2. **Autonomous vehicle navigation:** Understanding the motion of other vehicles, pedestrians, and obstacles is critical for self-driving cars and robots to plan safe paths.

3. **Human-computer interaction and gesture recognition:** Interpreting human body movements and hand gestures enables touchless interfaces, sign language recognition, and activity monitoring.

4. **Video compression and coding:** Motion estimation is fundamental to video codecs (e.g., MPEG, H.264) which encode motion vectors between frames rather than redundant pixel data, achieving significant compression.

5. **Biomedical and scientific analysis:** Tracking cell movement, cardiac motion analysis, and studying animal locomotion all rely on motion analysis techniques.

---

### (b) Three Reasons Why Motion Analysis is Difficult [3 marks]

1. **Illumination changes:** Variations in lighting between frames (shadows moving, lights turning on/off, clouds passing) cause pixel intensity changes unrelated to actual object motion, generating false motion signals.

2. **Occlusion:** Moving objects may partially or fully occlude each other or be occluded by static scene elements. This causes objects to appear/disappear suddenly, breaking temporal continuity assumptions.

3. **Complex/non-rigid motion:** Real-world objects undergo articulated motion (walking humans), deformation (facial expressions), or rotation, making it difficult to model with simple translational motion models. Additionally, camera motion introduces global motion that must be separated from object motion.

---

### (c) Burglar Detection in a Garage [9 marks]

#### (i) Two Problems with Simple Motion Detection [4 marks]

A "simple" motion detection approach (e.g., frame differencing — subtracting consecutive frames and thresholding) suffers from:

1. **Sensitivity to environmental changes:** Lighting fluctuations (car headlights passing, sunrise/sunset, flickering fluorescent lights), camera noise, and small irrelevant movements (leaves blowing, insects near the camera, rain) will all trigger false alarms. The garage environment may have automatic lights that turn on/off.

2. **Inability to detect stationary intruders or slow motion:** Frame differencing only detects pixels that change between consecutive frames. If a burglar stops moving (stands still), they become invisible to the detector. Similarly, very slow movement may produce differences below the threshold. A stopped burglar in the scene would not be detected until they move again.

---

#### (ii) Algorithm Addressing These Problems [5 marks]

**Background Subtraction with Adaptive Background Model:**

1. **Initialise background model:** Capture N frames of the empty garage. Compute the mean intensity μ(x,y) and standard deviation σ(x,y) for each pixel across these frames. This models the "normal" appearance including natural intensity variation.

2. **For each new frame:**
   - Compute the difference D(x,y) = |I(x,y) - μ(x,y)| for each pixel.
   - Classify pixel as foreground (potential intruder) if D(x,y) > T × σ(x,y), where T is a sensitivity parameter (e.g., T = 2.5). Using per-pixel σ accounts for areas of natural variation (e.g., flickering lights).

3. **Morphological post-processing:** Apply erosion to remove small noise regions (isolated false detections from camera noise), then dilation to fill gaps in detected objects and connect nearby foreground regions.

4. **Connected component analysis:** Group foreground pixels into connected blobs. Filter by minimum area — reject blobs too small to be a person (eliminates insects, small noise clusters).

5. **Temporal persistence check:** Require that a foreground region persists for multiple consecutive frames (e.g., 5-10 frames) before triggering an alarm. This eliminates transient false positives from brief lighting changes while detecting stationary intruders who remain in the scene.

6. **Adaptive background update:** Slowly update the background model over time: μ(x,y) ← α × I(x,y) + (1-α) × μ(x,y) for pixels classified as background. Use a very small learning rate α. This adapts to gradual illumination changes (day/night) without absorbing stationary intruders into the background (since foreground pixels are excluded from the update).

### (d) Motion Field vs Optical Flow [4 marks]

**Motion Field:** The motion field is the true 2D projection of 3D motion vectors onto the image plane. It represents the actual displacement of each visible surface point as projected into the camera. The motion field is a geometric/physical quantity — it describes where each 3D point actually moves to in the image.

**Optical Flow:** Optical flow is the apparent motion of brightness patterns in the image. It is computed from image intensity changes over time and is an estimate (approximation) of the motion field based on the assumption that pixel intensities are conserved during motion.

**Key differences:**

- They are not always equal. A uniformly coloured rotating sphere has a non-zero motion field (surface points are moving) but zero optical flow (the image appearance does not change because there are no texture variations to track).
- Conversely, a stationary scene with a moving light source has zero motion field (nothing is physically moving) but non-zero optical flow (brightness patterns shift due to changing illumination).
- Optical flow is what we can measure computationally; the motion field is the ground truth we wish to recover.

---

### (e) The Aperture Problem [5 marks]

**What it is:** The aperture problem arises when observing motion through a small local window (aperture). When viewing a moving edge through a limited aperture, only the component of motion perpendicular to the edge orientation can be detected. The component of motion parallel to the edge is invisible because the edge looks identical when shifted along its own direction.

For example, if a horizontal edge moves diagonally (both right and down), an observer looking through a small aperture sees only downward motion — the rightward component is indistinguishable because the edge pattern is uniform along its length.

Mathematically, the optical flow constraint equation:

Ix × u + Iy × v + It = 0

provides only one equation for two unknowns (u, v), where Ix, Iy are spatial gradients and It is the temporal gradient. This is inherently underdetermined at a single pixel — we can only determine the flow component in the gradient direction.

**How to address it:**

1. **Lucas-Kanade method (local approach):** Assume that flow is constant within a small neighbourhood window. Multiple pixels within the window each provide one constraint equation, forming an overdetermined system solved by least squares. This works well at corners and textured regions where gradients vary in direction.

2. **Horn-Schunck method (global approach):** Add a global smoothness constraint that penalises flow field variations. Formulate as an optimisation problem that minimises both the brightness constancy error and a regularisation term (sum of squared flow gradients). This propagates information from well-constrained regions (corners, textured areas) into ambiguous regions (edges, uniform areas) through the smoothness assumption.

3. **Multi-scale / coarse-to-fine approaches:** Compute flow at coarse resolution (where the aperture problem is less severe due to larger effective aperture) and refine at finer scales.

## Question 4: Tracking (25 marks)

### (a) Three Main Components of Visual Trackers [3 marks]

1. **Motion model (dynamics model):** Predicts the likely position and state of the target in the next frame based on previous states. This could be a constant velocity model, random walk, or more complex dynamics (e.g., Kalman filter prediction step). It constrains the search space for the tracker.

2. **Appearance model (observation model):** Describes what the target looks like, enabling the tracker to evaluate how well a candidate region matches the target. This could be a template, colour histogram, set of features, or a learned classifier.

3. **Tracking engine (search/inference mechanism):** The algorithm that combines the motion model's prediction with the appearance model's evaluation to find the best target location in the current frame. Examples include exhaustive search, gradient descent, mean-shift, or particle filtering.

---

### (b) Issues with Template/Patch Matching [4 marks]

1. **Appearance change over time:** The target's appearance changes due to pose variation, rotation, scale change, deformation, and illumination changes. A fixed template quickly becomes outdated and matching performance degrades.

2. **Template drift:** If the template is updated to adapt to appearance changes, small errors accumulate over time. The template gradually drifts away from the true target appearance, eventually causing the tracker to lose the target or lock onto the background.

3. **Computational cost of exhaustive search:** Searching the entire image for the best template match is expensive, especially for large templates or high-resolution images. The complexity scales with image size, template size, and the range of scales/rotations considered.

4. **Background clutter and distractors:** When other objects or background regions have similar appearance to the template, the matcher may produce multiple peaks or lock onto a distractor rather than the true target.

---

### (c) Four General Issues Faced by Trackers [6 marks]

1. **Occlusion:** The target may be partially or fully occluded by other objects. During partial occlusion, the appearance model receives corrupted observations. During full occlusion, the target is invisible and the tracker must maintain a prediction without observations, then re-acquire the target when it reappears.

2. **Scale and appearance variation:** As the target moves closer to or further from the camera, it changes size. Combined with 3D rotation, articulation (e.g., a walking person), and illumination changes, the target's appearance in the image can vary dramatically over time.

3. **Background clutter and similar objects:** When the background contains textures or colours similar to the target, or when multiple similar objects are present (e.g., tracking one person in a crowd), the tracker may confuse the target with distractors and "jump" to the wrong object.

4. **Fast or abrupt motion:** Rapid or unpredictable changes in velocity or direction violate the motion model assumptions. If the target moves further than the search window between frames, the tracker loses it. Motion blur from fast movement also degrades the appearance model's ability to match.

### (d) Active Contours (Snakes) Energy Function [8 marks]

An active contour (snake) is a deformable curve v(s) = (x(s), y(s)) for s in [0, 1] that evolves to minimise a total energy functional.

#### Internal Energy

The internal energy controls the shape properties of the contour itself:

E_internal(v) = α(s)|v'(s)|² + β(s)|v''(s)|²

- **First-order term** α(s)|v'(s)|²: Controls the **elasticity** (tension) of the contour. It penalises stretching, encouraging the contour to shrink and act like a rubber band. The parameter α controls how much the contour resists elongation.

- **Second-order term** β(s)|v''(s)|²: Controls the **rigidity** (stiffness) of the contour. It penalises bending/curvature, encouraging the contour to remain smooth. The parameter β controls how much the contour resists sharp corners.

#### External Energy

The external energy attracts the contour toward desired image features:

E_external(v) = -|∇I(v(s))|²

This is typically defined as the negative of the squared image gradient magnitude. The contour is attracted to strong edges (high gradient regions). Other formulations include:

- E_external = -|∇(Gσ * I)|² (gradient of Gaussian-smoothed image for wider capture range)
- Energy terms based on image intensity for attracting to dark/bright lines

#### Complete Energy Function

The total energy to be minimised is:

E_total = ∫₀¹ [E_internal(v(s)) + E_external(v(s))] ds

= ∫₀¹ [α(s)|v'(s)|² + β(s)|v''(s)|² + E_external(v(s))] ds

#### How It Guides Tracking

For tracking, the active contour from the previous frame provides the initial contour position in the current frame. The contour then evolves (iteratively moves its control points) to minimise the total energy in the new frame:

1. The internal energy maintains temporal coherence by resisting dramatic shape changes between frames.
2. The external energy pulls the contour toward the new boundary of the target object.
3. The balance between internal and external energies means the contour adapts to the object's new position and shape while maintaining smooth, plausible deformations.
4. Iterative minimisation (e.g., via Euler-Lagrange equations discretised with finite differences) moves each contour point according to internal forces (smoothing) and external forces (edge attraction) until equilibrium is reached.

---

### (e) Modifying Active Contours for Different Corner Properties [4 marks]

To track shapes with different corner properties (e.g., a sharp-cornered triangle vs a smoother pentagon), we modify the **rigidity parameter β(s)** along the contour:

**For a triangle (sharp corners):**
- Set β(s) = 0 (or very small) at the three corner points of the contour. This removes the bending penalty at those locations, allowing the contour to form sharp corners (high curvature) at those points.
- Maintain normal β values along the straight edges between corners to keep those segments smooth and straight.

**For a pentagon (moderate corners):**
- Set β(s) to a small but non-zero value at the five corner points. This allows moderate bending but still penalises very sharp angles, producing the less acute corners characteristic of a pentagon.
- Higher β along edges keeps them smooth.

**Implementation approach:**
- Use prior knowledge of the target shape to define a "prototype" contour with known corner locations.
- Assign low β values at s-positions corresponding to corners and high β values at positions corresponding to smooth segments.
- Alternatively, learn the β(s) function from training examples of the target shape.
- The number of low-β points determines the number of corners (3 for triangle, 5 for pentagon), and the magnitude of β at those points controls corner sharpness.

This makes the snake "aware" of the expected shape topology, allowing it to snap to the correct shape during tracking rather than smoothing out all corners uniformly.

## Question 5: Event Detection / Hidden Markov Models (25 marks)

### (a) "Hidden" and "Markov" in HMM [4 marks]

**"Hidden":** The states of the system are not directly observable. We can only observe outputs (emissions/observations) that are probabilistically related to the underlying states. For example, in activity recognition, we observe image features but the actual activity (the state) must be inferred — it is hidden from direct measurement. We must deduce the state sequence from the observation sequence.

**"Markov":** The system satisfies the Markov property (memorylessness): the probability of transitioning to the next state depends only on the current state, not on the sequence of states that preceded it. Formally:

P(qt+1 | qt, qt-1, ..., q1) = P(qt+1 | qt)

This first-order Markov assumption makes the model computationally tractable while still capturing sequential dependencies between states.

---

### (b) Two Sets and Three Functions of an HMM [5 marks]

**Two sets:**

1. **Set of states S = {s1, s2, ..., sN}:** The N possible hidden states the system can be in at any time step. These represent the underlying conditions we wish to infer.

2. **Set of observation symbols V = {v1, v2, ..., vM}:** The M possible observable outputs that can be emitted at each time step. These are what we actually measure.

**Three functions (probability distributions):**

1. **State transition probability matrix A = {aij}:** where aij = P(qt+1 = sj | qt = si). This defines the probability of moving from state i to state j. It is an N×N matrix where each row sums to 1.

2. **Observation/emission probability matrix B = {bj(k)}:** where bj(k) = P(ot = vk | qt = sj). This defines the probability of observing symbol vk when the system is in state sj. It is an N×M matrix where each row sums to 1.

3. **Initial state distribution π = {πi}:** where πi = P(q1 = si). This defines the probability of starting in each state. It is a vector of length N that sums to 1.

The complete HMM is specified as λ = (A, B, π).

---

### (c) Three Questions HMMs Allow Us to Ask [3 marks]

1. **Evaluation (Likelihood):** Given a model λ and an observation sequence O = (o1, o2, ..., oT), what is the probability P(O|λ) that the model generated those observations? (Solved by the Forward algorithm.) This is useful for classification — choose the model that best explains the observations.

2. **Decoding (State estimation):** Given a model λ and an observation sequence O, what is the most likely sequence of hidden states Q = (q1, q2, ..., qT) that produced the observations? (Solved by the Viterbi algorithm.) This reveals the underlying state sequence — e.g., what activities occurred.

3. **Learning (Training):** Given observation sequences, how do we adjust the model parameters λ = (A, B, π) to maximise P(O|λ)? (Solved by the Baum-Welch / Forward-Backward algorithm, a special case of EM.) This trains the model from data.

### (e) Using HMMs to Detect Suspicious Behaviour [7 marks]

**Approach:**

The key insight is to train separate HMM models for normal and suspicious behaviours, then use the evaluation capability to detect anomalies.

**Step 1: Define observations**

From video surveillance, extract observation features at each time step, such as:
- Speed of movement (slow, medium, fast)
- Direction changes (none, slight, sharp)
- Proximity to cars (near, far)
- Time spent stationary (brief, long)
- Location in car park (driving lane, between cars, near exit)

Quantise these into discrete observation symbols.

**Step 2: Train normal behaviour models**

Collect many observation sequences of normal car park behaviour (person arrives, walks to car, enters car, drives away; or arrives by car, parks, exits, walks away). Train an HMM λ_normal using the Baum-Welch algorithm on these sequences to learn typical transition and emission probabilities.

**Step 3: Detection by likelihood thresholding**

For a new observation sequence O of a person in the car park:
1. Compute P(O | λ_normal) using the Forward algorithm.
2. If P(O | λ_normal) < threshold, the behaviour is unlikely under the normal model — flag as **suspicious**.

**Step 4: Characterise suspicious patterns**

Suspicious behaviours in a car park might include:
- Prolonged standing near cars without entering any (possible car theft reconnaissance)
- Repeatedly walking between many cars (checking for unlocked doors)
- Entering/exiting multiple cars in sequence
- Walking-standing-walking pattern near many different vehicles

These would produce observation sequences with low probability under the normal model because normal people go directly to their own car.

**Step 5: Optionally train suspicious models**

If examples of suspicious behaviour are available, train a separate λ_suspicious model. Then classify using a likelihood ratio:
- If P(O | λ_suspicious) / P(O | λ_normal) > threshold → suspicious behaviour detected.

**Step 6: Viterbi decoding for explanation**

Once suspicious behaviour is flagged, run the Viterbi algorithm to find the most likely state sequence. This explains *what* the person was doing (e.g., "walked, stood near car A, walked, stood near car B, ...") providing interpretable output for security personnel.

**Advantages of HMMs for this task:**
- Naturally handle temporal sequences of varying length
- Can model the sequential structure of activities
- Provide probabilistic confidence measures
- The hidden states capture meaningful activity labels even when observations are noisy
