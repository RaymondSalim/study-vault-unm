## Question 1: Image Features [13 marks]

### (a) Main Steps to Obtain HOG Feature Representation [5 marks]

The Histogram of Oriented Gradients (HOG) descriptor captures the distribution of local edge directions in an image patch. The main steps are:

**Step 1: Gradient Computation**
Compute the horizontal and vertical gradients at each pixel using finite difference filters (e.g., [-1, 0, 1] and [-1, 0, 1]^T). From these, calculate the gradient magnitude and orientation at each pixel:

- Magnitude: m(x, y) = sqrt(Ix^2 + Iy^2)
- Orientation: theta(x, y) = arctan(Iy / Ix)

**Step 2: Cell Division**
Divide the image patch into small spatial regions called cells (typically 8x8 pixels each).

**Step 3: Orientation Binning within Cells**
For each cell, create a histogram of gradient orientations (typically 9 bins covering 0-180 degrees for unsigned gradients). Each pixel in the cell contributes a vote to the histogram bin corresponding to its gradient orientation, weighted by its gradient magnitude. Votes can be interpolated between neighbouring bins.

**Step 4: Block Normalisation**
Group adjacent cells into larger blocks (typically 2x2 cells per block). Concatenate the histograms of all cells within a block and normalise the resulting vector (e.g., using L2-norm: v / sqrt(||v||^2 + epsilon)). Blocks typically overlap, so each cell contributes to multiple blocks.

**Step 5: Feature Vector Assembly**
Concatenate all normalised block histograms across the entire detection window to form the final HOG feature vector. For a typical 64x128 detection window with 8x8 cells, 2x2 blocks, 1-cell stride, and 9 bins, this yields a vector of length 3780.

The block normalisation step is crucial as it provides invariance to local illumination changes and contrast variations.

### (c) SIFT Key Point Detection

**Therefore, the Hessian (structure tensor) matrix is:**

```

    [  4  63 ]
```

#### (ii) How the Hessian Matrix Determines a Key Point [2 marks]

A point is classified as a key point (corner/interest point) based on the eigenvalues of the Hessian matrix H:

- If both eigenvalues (lambda_1, lambda_2) are large, the point has significant gradient variation in both directions, indicating a **corner** (key point).
- If one eigenvalue is large and the other is small, the point lies on an **edge** (not a key point).
- If both eigenvalues are small, the region is **flat** (not a key point).

In practice, computing eigenvalues directly is avoided. Instead, the Harris corner response function is used:

R = det(H) - k * trace(H)^2

where det(H) = lambda_1 * lambda_2 and trace(H) = lambda_1 + lambda_2. A point is a key point if R exceeds a threshold (both eigenvalues are large, giving a large determinant relative to the trace).

For our matrix: det(H) = 73 * 63 - 4 * 4 = 4599 - 16 = 4583, trace(H) = 73 + 63 = 136. With k = 0.04: R = 4583 - 0.04 * 136^2 = 4583 - 739.84 = 3843.16. This large positive R indicates a strong key point.

### (b) Epipolar Geometry

#### (i) Epipolar Plane [2 marks]

The **epipolar plane** is the plane formed by three points: the 3D scene point P and the two camera centres O1 and O2. Referring to the figure, the triangle O1-P-O2 defines this plane. Every 3D point P produces a different epipolar plane, but all epipolar planes share the common baseline O1-O2 (the line connecting the two camera centres).

The epipolar plane intersects each image plane along a line called the **epipolar line**. In the left image, this is the line through p and e (the epipole), and in the right image, it is the line through p' and e' (the epipole in the right image).

#### (ii) Concept of Epipolar Geometry and Derivation of p^T F p' = 0 [4 marks]

**Epipolar geometry** describes the geometric relationship between two views of the same scene. It encodes the constraint that corresponding points in two images must satisfy, without needing to know the 3D structure of the scene.

**Key concepts:**
- **Epipoles (e, e'):** The projection of one camera centre onto the other camera's image plane. e is the image of O2 in camera 1; e' is the image of O1 in camera 2.
- **Epipolar lines:** For a point p in the left image, its corresponding point p' must lie on a specific line (the epipolar line) in the right image. This is because the ray from O1 through p defines a line in 3D space, which projects to a line in the right image.
- **Epipolar constraint:** This reduces the correspondence search from 2D to 1D.

**Derivation of the Fundamental Matrix equation:**

A 3D point P projects to p in image 1 and p' in image 2. The vectors O1P (represented by p in camera 1 coordinates) and the baseline O1O2 (translation t) and the rotated point in camera 2 (Rp' back to camera 1 frame) must all be coplanar (they lie in the epipolar plane).

The coplanarity condition states that the scalar triple product must be zero:

p'^T * [t]_x * R * p = 0

where [t]_x is the skew-symmetric matrix of the translation vector. Defining the **Essential Matrix** E = [t]_x * R:

p'^T * E * p = 0 (in normalised camera coordinates)

Incorporating the intrinsic matrices K and K' that map pixel coordinates to normalised coordinates (p_normalised = K^{-1} * p_pixel):

(K'^{-1} p')^T * E * (K^{-1} p) = 0

p'^T * K'^{-T} * E * K^{-1} * p = 0

Defining the **Fundamental Matrix** F = K'^{-T} * E * K^{-1}:

**p'^T F p = 0**

This is the fundamental epipolar constraint. F is a 3x3 matrix of rank 2 that encapsulates the epipolar geometry between two views. For any corresponding pair of points (p, p') in pixel coordinates, this equation must hold. The line Fp in the right image is the epipolar line on which p' must lie, and F^T p' in the left image is the epipolar line on which p must lie.

**Output dimensions: 64 channels x 29 x 29**

Note: The number of input channels (16) affects the filter depth (each of the 64 filters has shape 16 x 7 x 7), but does not affect the spatial output dimensions.

### (b) Differences Between 2D Convolution and 2D Pooling [4 marks]

| Aspect | 2D Convolution | 2D Pooling |
|--------|---------------|------------|
| **Parameters** | Has learnable parameters (filter weights and biases) that are trained via backpropagation | Has no learnable parameters; performs a fixed operation (max or average) |
| **Function** | Computes a weighted sum of the input values in the receptive field using learned filter weights, then applies an activation function. Extracts features such as edges, textures, and patterns | Applies a fixed aggregation function (max or average) over the receptive field to downsample the feature map and provide translational invariance |
| **Channel interaction** | Operates across all input channels simultaneously. A single filter has depth equal to the number of input channels and produces one output channel. Multiple filters produce multiple output channels | Operates on each channel independently. The number of channels remains unchanged after pooling |
| **Output size** | Can maintain spatial dimensions (with appropriate padding) or reduce them. Commonly used with padding to preserve dimensions | Primarily used to reduce spatial dimensions (downsample), typically by a factor of 2 with 2x2 pooling and stride 2 |

In summary, convolution is a learned feature extraction operation that can change the number of channels, while pooling is a fixed downsampling operation that reduces spatial resolution while preserving channel count. Convolution captures "what" features are present; pooling provides spatial compression and local invariance.

### (c) Vision Transformer (ViT) Architecture

#### (i) Attention Mechanism and Multi-Head Attention [4 marks]

**Attention Mechanism:**

The attention mechanism allows the model to dynamically focus on different parts of the input sequence when producing each output element. Given an input sequence of patch embeddings, the self-attention mechanism computes:

1. **Query (Q), Key (K), Value (V) projections:** Each input token is linearly projected into three vectors: a query, a key, and a value using learned weight matrices (W_Q, W_K, W_V).

2. **Attention scores:** The compatibility between tokens is computed as the scaled dot product of queries and keys:
   Attention(Q, K, V) = softmax(Q * K^T / sqrt(d_k)) * V

   where d_k is the dimension of the key vectors (scaling prevents dot products from becoming too large).

3. **Weighted aggregation:** The softmax output provides attention weights that determine how much each token attends to every other token. The output for each token is a weighted sum of all value vectors.

This allows each patch to gather information from all other patches in the image, capturing long-range dependencies that CNNs struggle with.

**Multi-Head Attention:**

Multi-head attention runs multiple attention operations in parallel, each with different learned projections:

MultiHead(Q, K, V) = Concat(head_1, ..., head_h) * W_O

where head_i = Attention(Q * W_Qi, K * W_Ki, V * W_Vi)

Each head can learn to attend to different aspects of the input (e.g., one head might capture spatial relationships, another might focus on colour similarity, another on texture patterns). The outputs of all heads are concatenated and linearly projected to produce the final output. This allows the model to jointly attend to information from different representation subspaces at different positions.

#### (ii) Class Token: Definition, Purpose, and Operation [4 marks]

**What it is:**

The class token (often denoted [CLS]) is a special learnable embedding vector that is prepended to the sequence of patch embeddings before they enter the transformer encoder. It is not derived from any image patch but is a trainable parameter of the model, initialised randomly and updated during training via backpropagation.

**Its purpose:**

The class token serves as a global representation aggregator for the entire image. Its role is to accumulate information from all patch tokens through the self-attention layers and ultimately provide a single fixed-size vector that represents the entire image for classification.

**How it works for classification:**

1. **Input construction:** The image is divided into fixed-size patches (e.g., 16x16), each patch is linearly embedded, positional embeddings are added, and the [CLS] token is prepended to form the input sequence of length (N+1), where N is the number of patches.

2. **Information aggregation:** As the sequence passes through multiple transformer encoder layers, the [CLS] token attends to all patch tokens (and vice versa) via self-attention. Through successive layers, the [CLS] token progressively integrates global information from all image patches.

3. **Classification head:** After the final transformer encoder layer, only the output corresponding to the [CLS] token position is extracted. This single vector is fed into a classification MLP head (typically one or two linear layers with a softmax) to produce class probabilities.

The [CLS] token is analogous to global average pooling in CNNs -- it produces a single feature vector summarising the entire input -- but it achieves this through learned attention rather than a fixed averaging operation, allowing more flexible and content-dependent aggregation.

## Question 4: Image Segmentation and Optical Flow [30 marks]

### (a) GMM-Based Segmentation of Reindeer Image [16 marks]

**Objective:** Segment the image into three classes: reindeer, grass, and woods using a Gaussian Mixture Model (GMM) in HSV colour space.

**Choice of Colour Space -- HSV Justification:**

HSV separates chromatic information (Hue, Saturation) from intensity (Value), making it more robust to illumination variations than RGB. The three target objects (reindeer, grass, woods) differ primarily in their colour properties:
- **Reindeer:** Brown/tan colour -- Hue around 20-30, moderate-high saturation, moderate value
- **Golden grass:** Similar hue range (20-40) but different saturation and value characteristics
- **Woods:** Green/dark tones -- Hue around 60-120 (green), lower value (darker), variable saturation

**Analysis of Histograms:**

From the described histograms:
- **Hue histogram:** A large peak around 20-30 corresponds to both the golden grass and brown reindeer (warm colours). A secondary distribution at higher hue values likely corresponds to the green woods.
- **Saturation histogram:** Peaks around 50-100 suggest moderate saturation for natural outdoor colours. Reindeer fur tends to be more saturated than dried grass; woods may have variable saturation.
- **Value histogram:** Peaks around 50-100 (darker regions: woods/shadows) and 150-200 (brighter regions: sunlit grass and reindeer). This bimodality helps distinguish woods (darker) from grass and reindeer (brighter).

**GMM Design:**

**Number of components:** Use g = 3 Gaussian components, one for each target class (reindeer, grass, woods). Alternatively, g = 4 or 5 may be used if sub-classes exist (e.g., sunlit vs. shadowed grass), but 3 is the minimum and aligns with the segmentation target.

**Feature vector:** Each pixel is represented as a 3D vector x = [H, S, V]. Using all three channels provides maximum discriminative power:
- H distinguishes woods (green hues) from reindeer and grass (warm hues)
- S helps separate reindeer (higher saturation) from grass (lower saturation for dried/golden grass)
- V helps separate woods (darker) from grass and reindeer (brighter)

**GMM Parameters:**
Each component k has parameters theta_k = {mu_k, Sigma_k, pi_k}:
- mu_k: 3D mean vector [mean_H, mean_S, mean_V] for class k
- Sigma_k: 3x3 covariance matrix (using full covariance to capture correlations between H, S, V channels)
- pi_k: mixing coefficient (prior probability of class k)

**Training via EM Algorithm:**

1. **Initialisation:** Use k-means clustering on the HSV pixel values to initialise the means. Set initial covariances to the sample covariance within each cluster. Set initial mixing coefficients proportional to cluster sizes.

2. **E-Step:** Compute posterior probabilities (responsibilities) for each pixel:
   r_ik = (pi_k * N(x_i | mu_k, Sigma_k)) / sum_j(pi_j * N(x_i | mu_j, Sigma_j))

3. **M-Step:** Update parameters:
   - mu_k = sum_i(r_ik * x_i) / sum_i(r_ik)
   - Sigma_k = sum_i(r_ik * (x_i - mu_k)(x_i - mu_k)^T) / sum_i(r_ik)
   - pi_k = (1/N) * sum_i(r_ik)

4. **Convergence:** Repeat E and M steps until log-likelihood stabilises.

5. **Segmentation:** Assign each pixel to the class with highest posterior: class(x_i) = argmax_k r_ik.

**Discussion and Challenges:**

- **Overlapping hues:** The reindeer and golden grass have very similar hue values (both in the 20-30 range), making hue alone insufficient for separating them. The combination of saturation and value is critical for this distinction.
- **Full covariance:** Using full 3x3 covariance matrices (rather than diagonal) allows modelling of correlations between channels. For example, brighter grass pixels (high V) may also tend to be more saturated.
- **Potential improvements:** Spatial information (e.g., Markov Random Field prior) could help enforce spatial coherence in the segmentation, as the reindeer forms a connected region distinct from surrounding grass. Post-processing with morphological operations could clean up noisy classifications.
- **Limitations:** GMMs assume pixel-wise independence and do not model spatial context. Pixels in shadowed areas may be misclassified. The similar colour of reindeer and dried grass is the primary challenge.

## Question 5: Stereo Vision [30 marks]

### (a) Estimating Three Dimensions from Stereo Images [18 marks]

**Overview:**

Stereo vision allows reconstruction of 3D geometry from two (or more) images taken from different viewpoints. Given a rectified stereo pair showing packages/cargo with four labelled corresponding points, we can estimate the three physical dimensions of the cargo (108 cm, 139 cm, 159 cm) by reconstructing 3D coordinates and computing Euclidean distances.

**Step 1: Disparity Computation**

For a rectified stereo pair (where epipolar lines are horizontal), the **disparity** d for a point is defined as:

d = x_L - x_R

where x_L is the horizontal pixel coordinate in the left image and x_R is the corresponding horizontal coordinate in the right image. For rectified images, corresponding points share the same vertical coordinate, so the search for correspondences is along horizontal scanlines only.

A larger disparity indicates a closer point (the point shifts more between views when it is nearer to the cameras).

**Step 2: Depth from Disparity**

The relationship between disparity and depth is derived from similar triangles in the stereo geometry:

**Z = f * B / d**

where:
- Z is the depth (distance from the camera baseline to the 3D point)
- f is the focal length (in pixels)
- B is the baseline (physical distance between the two camera centres)
- d is the disparity (in pixels)

This inverse relationship means that depth resolution decreases quadratically with distance -- nearby objects have large disparities and precise depth estimates, while distant objects have small disparities and coarser depth.

**Step 3: 3D Point Reconstruction**

Once depth Z is known for a pixel (u, v) in the left image, the full 3D coordinates (X, Y, Z) in the left camera's coordinate frame can be recovered:

X = (u - cx) * Z / f
Y = (v - cy) * Z / f
Z = f * B / d

where (cx, cy) is the principal point of the left camera. This maps each pixel with a known disparity to a unique point in 3D space.

For each of the four labelled points (Origin, 108 cm, 139 cm, 159 cm), we:
1. Find the corresponding point in the right image
2. Compute disparity d = x_L - x_R
3. Compute depth Z = f * B / d
4. Reconstruct 3D coordinates (X, Y, Z) using the equations above

**Step 4: Euclidean Distance for Physical Dimensions**

The three dimensions of the cargo are estimated by computing the **Euclidean distance** between pairs of reconstructed 3D points:

Distance = sqrt((X2 - X1)^2 + (Y2 - Y1)^2 + (Z2 - Z1)^2)

For the cargo dimensions:
- **108 cm:** Euclidean distance between the "Origin" point and the "108 cm" point in 3D
- **139 cm:** Euclidean distance between the "Origin" point (or appropriate corner) and the "139 cm" point
- **159 cm:** Euclidean distance between the "Origin" point (or appropriate corner) and the "159 cm" point

These three measurements correspond to the width, height, and depth of the cargo package.

**Practical Considerations:**

- **Rectification:** The images must be rectified so that epipolar lines are horizontal, simplifying correspondence search. This is achieved by applying homographies derived from the stereo calibration.
- **Calibration:** The camera parameters (f, cx, cy, B) must be known from prior calibration (e.g., using a checkerboard pattern). Without accurate calibration, metric measurements are impossible.
- **Correspondence accuracy:** The accuracy of the 3D measurements depends critically on precise point correspondences. Sub-pixel accuracy in correspondence leads to sub-pixel disparity and more accurate depth.
- **Baseline-depth trade-off:** A larger baseline B increases depth resolution but also increases occlusion regions where one camera sees a surface that the other cannot.

**Worked Example Structure:**

If the four points have pixel coordinates:
- Origin: (u1_L, v1_L) in left, (u1_R, v1_R) in right => d1 = u1_L - u1_R
- Point A (108 cm away): similarly => d2, then Z2 = fB/d2
- Reconstruct both in 3D, compute: sqrt((X_A - X_O)^2 + (Y_A - Y_O)^2 + (Z_A - Z_O)^2) = 108 cm

This demonstrates how stereo vision converts 2D image measurements into accurate 3D metric measurements.

### (b) Deep Learning Stereo Pipeline [12 marks]

The deep learning stereo pipeline consists of five stages:

**(A) Dense/Sparse Feature Extraction**

A deep CNN (e.g., a series of convolutional layers with batch normalisation and ReLU activations) extracts feature representations from both the left and right images independently. The same network (shared weights, Siamese architecture) processes both images to ensure consistent feature representations.

- **Dense features:** Every pixel receives a feature descriptor (a high-dimensional vector). This is achieved using fully convolutional networks that maintain spatial resolution (or use encoder-decoder structures). Dense features enable computation of a complete disparity map.
- **Sparse features:** Features are extracted only at salient locations (e.g., corners, edges). This is faster but produces only sparse depth estimates.

The extracted features are more discriminative than raw pixel intensities, being robust to illumination changes, noise, and local appearance variations. Typical architectures use multi-scale feature extraction to capture both fine details and broader context.

**(B) Feature Matching**

Features from the left image are matched with features from the right image along epipolar lines (horizontal scanlines for rectified images). For each pixel in the left image, its feature vector is compared with feature vectors at all possible disparities in the right image.

A **cost volume** is constructed: a 3D tensor of dimensions H x W x D_max, where D_max is the maximum disparity. Each entry (x, y, d) stores the matching cost (e.g., correlation, concatenation, or absolute difference) between the left feature at (x, y) and the right feature at (x-d, y).

Common approaches:
- **Correlation-based:** Compute dot product or cosine similarity between feature vectors
- **Concatenation-based:** Concatenate left and right features and use subsequent layers to learn the matching function
- **Group-wise correlation:** Split features into groups and compute correlation within each group (GwcNet)

**(C) Cost Volume Regularisation**

Raw matching costs are noisy and may contain ambiguities (e.g., in textureless regions or repetitive patterns). Cost volume regularisation applies 3D convolutions (or other operations) to the cost volume to enforce spatial smoothness and resolve ambiguities.

The 3D CNN processes the H x W x D_max volume, applying learned filters that:
- Aggregate contextual information from neighbouring pixels and disparities
- Smooth out noise and outliers
- Enforce the constraint that disparity varies smoothly across surfaces
- Use hourglass or encoder-decoder architectures with skip connections to capture multi-scale context

This stage replaces the hand-crafted smoothness penalties (e.g., semi-global matching penalties) used in traditional methods with learned regularisation that adapts to the data.

**(D) Obtain Raw Disparity Map (Argmin)**

From the regularised cost volume, the disparity for each pixel is determined by selecting the disparity with the lowest cost (or highest correlation):

d(x, y) = argmin_d C(x, y, d)

In practice, a **soft argmin** (differentiable) is often used to enable end-to-end training:

d(x, y) = sum_d (d * softmax(-C(x, y, d)))

This produces a continuous (sub-pixel) disparity estimate rather than a discrete integer value, and allows gradients to flow through this operation during backpropagation. The result is a dense disparity map at the resolution of the input images.

**(E) Post-Processing and Refinement**

The raw disparity map may contain errors, particularly at object boundaries, occluded regions, and textureless areas. Post-processing refines the output:

- **Context network / residual refinement:** Additional CNN layers take the raw disparity map (and possibly the left image) as input and predict a residual correction, sharpening boundaries and correcting errors.
- **Left-right consistency check:** Compute disparity maps from both left-to-right and right-to-left. Pixels where the two maps disagree by more than a threshold are marked as occluded or unreliable.
- **Edge-aware filtering:** Apply bilateral or guided filtering to align disparity edges with image edges.
- **Upsampling:** If the cost volume was computed at a reduced resolution (for efficiency), learned upsampling recovers full-resolution disparity.
- **Confidence estimation:** Some networks output a confidence map alongside the disparity, indicating reliability of each estimate.

The entire pipeline (A through E) can be trained end-to-end using supervised learning with ground truth disparity maps (e.g., from LiDAR or structured light), or self-supervised using photometric consistency losses.

### (a) Viola-Jones Integral Image [10 marks]

**Given 3x3 patch:**
```
90   95   91
85  120  100
50   70   60
```

**Calculate the Integral Image:**

The integral image II(x, y) stores the sum of all pixel values above and to the left of (x, y), inclusive:

II(x, y) = sum_{x' <= x, y' <= y} I(x', y')

Using the recurrence: II(x, y) = I(x, y) + II(x-1, y) + II(x, y-1) - II(x-1, y-1)

With zero-padding on the top and left borders:

Row 0:
- II(0,0) = 90
- II(1,0) = 90 + 95 = 185
- II(2,0) = 185 + 91 = 276

Row 1:
- II(0,1) = 90 + 85 = 175
- II(1,1) = 120 + 175 + 185 - 90 = 390
- II(2,1) = 100 + 276 + 390 - 185 = 581

Row 2:
- II(0,2) = 175 + 50 = 225
- II(1,2) = 70 + 225 + 390 - 175 = 510
- II(2,2) = 60 + 581 + 510 - 390 = 761

**Integral Image:**
```
 90  185  276
175  390  581
225  510  761
```

**Calculate the Simple Haar-like Feature (Vertical Two-Rectangle):**

A vertical two-rectangle feature with left dark and right light divides the patch into left and right halves. We subtract the sum of the light (right) region from the sum of the dark (left) region.

For a 3x3 patch split into left (column 0, and half of column 1) and right (half of column 1, column 2):

Alternatively, taking the simplest split -- left column vs right column for a 2-column feature within the patch:

**Left rectangle (column 0, all rows):** 90 + 85 + 50 = 225
**Right rectangle (column 1, all rows):** 95 + 120 + 70 = 285

Using integral image to compute rectangle sums efficiently:
- Sum of left column = II(0,2) = 225 (directly, since it's the leftmost column from top to bottom)
- Sum of middle column = II(1,2) - II(0,2) = 510 - 225 = 285

For a two-rectangle feature (left dark minus right light):
Feature value = Sum(dark/left) - Sum(light/right) = 225 - 285 = **-60**

Alternatively, if the feature spans the full 3x3 with left half (columns 0-1) and right half (column 2):
- Sum of left half (cols 0-1, rows 0-2) = II(1,2) = 510
- Sum of right half (col 2, rows 0-2) = II(2,2) - II(1,2) = 761 - 510 = 251

Feature value = 510 - 251 = **259**

The specific feature value depends on the exact rectangle positions and sizes chosen.

Using the integral image, any rectangular sum can be computed in exactly 4 array lookups regardless of rectangle size:

Sum(rectangle) = II(x2, y2) - II(x1-1, y2) - II(x2, y1-1) + II(x1-1, y1-1)

**How to Generate Multiple Features:**

The Viola-Jones framework generates a large pool of candidate features by varying:

1. **Feature type:** Different templates (two-rectangle horizontal, two-rectangle vertical, three-rectangle, four-rectangle/checkerboard patterns). Each captures different image structures (edges, lines, centre-surround patterns).

2. **Position:** The feature can be placed at any (x, y) position within the detection window (e.g., 24x24 pixels).

3. **Scale/Size:** The rectangles can have any width and height (subject to fitting within the detection window). Both the overall feature size and the relative proportions of sub-rectangles can vary.

4. **Exhaustive enumeration:** All combinations of type, position, and size within the detection window are generated. For a 24x24 window, this produces approximately 160,000+ features.

5. **Feature selection:** AdaBoost is used to select the most discriminative features from this overcomplete set. Each weak classifier uses a single Haar-like feature with an optimal threshold. The boosting process iteratively selects features that best classify the weighted training examples, building a strong classifier from many weak ones.

6. **Cascade structure:** Features are organised into a cascade of increasingly complex stages. Early stages use few features to quickly reject obvious non-face regions; later stages use more features for difficult cases. This achieves high detection speed (most sub-windows are rejected early).

### (b) 3D Model Reconstruction with GAN [20 marks]

**GAN Concept:**

A **Generative Adversarial Network (GAN)** consists of two neural networks trained simultaneously in a minimax game:

1. **Generator (G):** Takes a random noise vector z (sampled from a prior distribution, e.g., standard normal) and produces synthetic data samples (in this case, 3D models). The generator learns to map from a latent space to the data distribution.

2. **Discriminator (D):** Takes a data sample (either real from the training set or fake from the generator) and outputs a probability that the sample is real. The discriminator learns to distinguish real from generated samples.

**Training objective (minimax game):**

min_G max_D V(D, G) = E_{x~p_data}[log D(x)] + E_{z~p_z}[log(1 - D(G(z)))]

- The discriminator maximises V by correctly classifying real and fake samples
- The generator minimises V by producing samples that fool the discriminator
- At equilibrium, G generates samples indistinguishable from real data, and D outputs 0.5 for all inputs

**Application to 3D Reconstruction from 2D Input:**

The described framework extends the basic GAN to perform conditional 3D reconstruction from a single 2D image. The architecture consists of:

**Encoder (2D Image to Latent Code):**

The encoder is a CNN that takes a 2D input image and compresses it into a compact latent representation Z. This latent code captures the essential shape, pose, and appearance information needed to reconstruct the corresponding 3D model. The encoder learns to extract 3D-relevant features from 2D observations, effectively learning an inverse rendering function.

**Class Label (C):**

A class label C is provided alongside the latent code. This conditions the generation on the object category (e.g., car, chair, airplane). The class information helps the generator:
- Focus on category-specific shape priors
- Resolve ambiguities where the same 2D silhouette could correspond to different 3D objects
- Share common structural knowledge within a class while allowing instance-specific variation

**Generator (Latent Code + Class to 3D Model):**

The generator takes the concatenated latent code Z and class label C as input and produces a 3D model. The 3D representation could be:
- A voxel grid (3D occupancy grid)
- A point cloud
- A mesh with vertex positions
- An implicit function (e.g., signed distance field)

The generator uses 3D transposed convolutions (for voxel output) or fully connected layers followed by reshaping to produce the 3D output. It learns the mapping from the compact 2D-derived representation to full 3D geometry.

**Classifier (Verification):**

A classifier network takes the generated 3D model and predicts its class label C'. This serves as an auxiliary loss that ensures the generated 3D model is consistent with the specified class. The classifier loss:
- Encourages the generator to produce class-consistent outputs
- Provides gradient signal about the semantic content of the generated shape
- Acts as a regulariser preventing mode collapse within classes

**Discriminator (Real vs. Generated):**

The discriminator evaluates whether a 3D model is real (from a dataset of ground truth 3D models) or generated. It provides the adversarial training signal that pushes the generator toward realistic outputs. The discriminator:
- Learns what constitutes realistic 3D geometry (smooth surfaces, plausible proportions, structural coherence)
- Can be conditioned on the class label for class-specific realism assessment
- May operate on the voxel grid, point cloud, or rendered views of the 3D model

**Training Process:**

1. **Input:** A 2D image of an object with known class label and corresponding ground truth 3D model (from a paired dataset like ShapeNet).

2. **Forward pass:**
   - Encoder processes 2D image to produce latent code Z
   - Z is concatenated with class label C
   - Generator produces 3D model G(Z, C)
   - Discriminator evaluates D(G(Z, C)) vs D(real 3D model)
   - Classifier predicts class C' from G(Z, C)

3. **Loss functions (multi-objective):**
   - **Adversarial loss:** Standard GAN loss ensuring realistic 3D outputs
   - **Reconstruction loss:** L1 or L2 distance between generated and ground truth 3D models (when paired data is available). This provides direct supervision.
   - **Classification loss:** Cross-entropy between predicted class C' and true class C
   - **Optional perceptual loss:** Render the 3D model from multiple viewpoints and compare with 2D images

4. **Optimisation:** Alternating updates:
   - Update discriminator to better distinguish real from fake
   - Update generator (and encoder) to fool discriminator, minimise reconstruction error, and satisfy classifier

**Inference (Test Time):**

At inference, the process is straightforward:
1. Input a novel 2D image (never seen during training) and its class label
2. The encoder maps it to latent code Z
3. The generator produces the 3D model from Z and C
4. No discriminator or classifier needed (they are training-time components only)

The trained encoder-generator pipeline thus acts as a feed-forward 3D reconstruction system that can produce a 3D model from a single 2D image in one pass (no optimisation or iterative refinement needed at test time).

**Advantages of the GAN-based approach:**

- The adversarial loss encourages globally coherent and realistic 3D shapes, beyond what pixel-wise reconstruction losses alone can achieve
- The generator learns a strong shape prior from the training data, enabling plausible reconstruction even from highly ambiguous single-view inputs
- The class conditioning allows a single model to handle multiple object categories while maintaining category-specific quality
- Once trained, inference is fast (single forward pass) compared to optimisation-based reconstruction methods

**Limitations:**
- Requires 3D ground truth data for training (or at minimum, multi-view images)
- Mode collapse may cause limited diversity in outputs
- Resolution of 3D outputs may be limited (especially for voxel representations)
- May struggle with object categories or viewpoints not well-represented in training data
