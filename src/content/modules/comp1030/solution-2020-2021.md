## Question 1: Networks (25 marks)

### (a) Three Types of Fiber-Optic Cables [5 marks]

There are three types of fiber-optic cables:

**1. Single-Mode Fiber (SMF)**

```
             Cladding
    ┌─────────────────────────────┐
    │  ┌───────────────────────┐  │
    │  │   → → → → → → → →   │  │  (very narrow core ~8-10μm)
    │  └───────────────────────┘  │
    └─────────────────────────────┘
         Single light ray path
```

- Has a very narrow core (8-10 micrometers).
- Light travels in a single straight path (one mode) directly through the core.
- Uses laser light source.
- Supports very long distances (up to 100+ km) with minimal signal loss.

**2. Multimode Step-Index Fiber**

```
             Cladding
    ┌─────────────────────────────┐
    │  ┌───────────────────────┐  │
    │  │  → ╲  ╱ ╲  ╱ ╲  ╱ → │  │  (wide core ~50-62.5μm)
    │  │  →   ╳    ╳    ╳   → │  │
    │  │  → ╱  ╲  ╱ ╲  ╱ ╲ → │  │
    │  └───────────────────────┘  │
    └─────────────────────────────┘
      Multiple rays bounce at angles
```

- Has a wider core (50-62.5 micrometers).
- Multiple light rays travel at different angles, bouncing off the cladding.
- The refractive index changes abruptly (step) between core and cladding.
- Causes modal dispersion (rays arrive at different times), limiting bandwidth and distance.
- Uses LED light source.

**3. Multimode Graded-Index Fiber**

```
             Cladding
    ┌─────────────────────────────┐
    │  ┌───────────────────────┐  │
    │  │    ⌒    ⌒    ⌒       │  │  (wide core, graded refractive index)
    │  │  →    →    →    →  → │  │
    │  │    ⌣    ⌣    ⌣       │  │
    │  └───────────────────────┘  │
    └─────────────────────────────┘
      Rays curve gradually back to center
```

- Has a wider core like step-index, but the refractive index gradually decreases from the center outward.
- Light rays curve back toward the center rather than bouncing sharply.
- Rays traveling longer paths move through lower refractive index regions (faster), so they arrive at roughly the same time.
- Reduces modal dispersion compared to step-index.
- Uses LED or laser light source.

**Most efficient: Single-Mode Fiber (SMF)**

Single-mode fiber is the most efficient because:
- It eliminates modal dispersion entirely (only one path exists).
- It achieves the highest bandwidth and longest transmission distances.
- It has the lowest signal attenuation.
- It supports the highest data rates (up to hundreds of Gbps over long distances).

### (b) CSMA/CD Protocols [6 marks]

#### (i) Non-persistent, 1-persistent, and p-persistent CSMA/CD [3 marks]

**Non-persistent CSMA/CD:**
- A station that wants to transmit senses the channel.
- If the channel is idle, the station transmits immediately.
- If the channel is busy, the station does NOT continuously sense the channel. Instead, it waits a random amount of time and then repeats the algorithm (senses the channel again).
- This reduces collisions but increases delay because the channel may go idle while the station is waiting.

**1-persistent CSMA/CD:**
- A station that wants to transmit senses the channel.
- If the channel is idle, the station transmits immediately (with probability 1, hence "1-persistent").
- If the channel is busy, the station continuously monitors the channel and transmits as soon as it becomes idle.
- If a collision occurs, the station waits a random time and starts over.
- This has high probability of collision when multiple stations are waiting because all will transmit simultaneously when the channel becomes free.

**p-persistent CSMA/CD:**
- A station that wants to transmit senses the channel.
- If the channel is idle, the station transmits with probability p, or defers to the next time slot with probability (1-p).
- If the channel is busy, the station waits until the next slot and applies the algorithm again.
- If a collision occurs, the station waits a random time and starts over.
- This is a compromise between non-persistent and 1-persistent.

#### (ii) Recommended algorithm for LAN [3 marks]

**Recommended: 1-persistent CSMA/CD** for LAN data transmission.

Reasons:
- In a LAN environment, propagation delays are very short (small geographic area), so the window for collisions is small.
- 1-persistent achieves the lowest delay under low-to-moderate traffic loads, which is typical of LAN usage.
- The immediate transmission when the channel is idle minimises waiting time.
- Although it has higher collision probability than p-persistent, the short propagation delay in LANs means the collision detection and recovery is fast.
- Ethernet (IEEE 802.3), the most widely used LAN technology, uses 1-persistent CSMA/CD.

### (c) Circuit Switching vs Virtual Circuit Switching [6 marks]

| Feature | Circuit Switching | Virtual Circuit Switching |
|---------|------------------|--------------------------|
| **Connection** | A dedicated physical path is established end-to-end before data transfer begins | A logical path is established; packets follow this predetermined route but share physical links with other traffic |
| **Resources** | Dedicated bandwidth reserved for the entire duration of communication (even during silence) | Resources are shared; bandwidth is allocated only when packets are actually being transmitted |
| **Setup** | Physical circuit setup required (takes time) | Virtual circuit setup required (routing decision made once) |
| **Data transfer** | Continuous stream of data (no addressing needed per unit) | Data divided into packets; each packet carries a virtual circuit identifier (VCI) |
| **Routing** | Fixed path; no per-unit routing decisions | Fixed path determined at setup; no per-packet routing decisions |
| **Overhead** | No per-packet overhead, but wastes bandwidth during idle periods | Small per-packet overhead (VCI), efficient bandwidth usage |
| **Order** | Data arrives in order | Packets arrive in order (same path) |
| **Examples** | Traditional telephone network (PSTN) | ATM networks, Frame Relay, MPLS |

**What happens when a node or link fails on a virtual circuit?**

When a node or communications link along a virtual path fails:
1. All virtual circuits passing through that node/link are broken and lost.
2. The network must detect the failure (via timeout or keep-alive messages).
3. All affected virtual circuits must be re-established — a new route must be computed and new VCI values assigned along the new path.
4. Any packets in transit on the failed portion are lost and must be retransmitted by higher-layer protocols (e.g., TCP).
5. Both endpoints must be notified of the failure so they can participate in re-establishment.
6. Unlike datagram switching, virtual circuits cannot automatically route around failures on a per-packet basis — the entire logical connection must be rebuilt.

### (d) Isolated Routing [4 marks]

**Isolated routing** (also called decentralised routing) is a routing approach where each node makes routing decisions based solely on its own local information, without exchanging routing information with other nodes. Each router independently determines the best path for forwarding packets.

**Methods of isolated routing include:**

1. **Hot potato (deflection routing):** A packet is forwarded to whichever outgoing link has the shortest queue, regardless of whether that link leads toward the destination.

2. **Backward learning:** A router learns routes by examining the source addresses of incoming packets and noting which interface they arrived on.

**How isolated routing may gather wrong or insufficient data:**

1. **No global view:** Since nodes do not exchange information, they have no knowledge of the overall network topology. A node cannot know about distant link failures, congestion, or topology changes until it directly observes effects.

2. **Stale information:** Backward learning derives routing information from traffic patterns. If traffic patterns change or a route changes, the node continues using old information until new packets arrive from the relevant source on a different interface.

3. **Asymmetric routes:** The path from A to B may differ from B to A. Backward learning assumes symmetric routes — if a packet from source X arrives on interface 1, it concludes X is reachable via interface 1. But the optimal forward path to X might be different.

4. **Incomplete information:** If no traffic has been received from a particular destination, the router has no routing information for that destination at all. Under light traffic conditions, routing tables remain incomplete.

5. **Congestion-based errors (hot potato):** Hot potato routing may send packets away from their destination simply because the correct queue is momentarily longer, potentially causing loops or excessive path lengths.

### (e) Collision Avoidance in CSMA/CA [4 marks]

**What is Collision Avoidance?**

Collision Avoidance is a mechanism designed to prevent collisions from occurring in the first place, rather than detecting them after they happen. It is necessary in wireless networks because collision detection (as in CSMA/CD) is impractical in wireless environments due to the hidden terminal problem and the fact that a station cannot transmit and listen simultaneously on the same frequency.

**How CSMA/CA achieves collision avoidance:**

1. **Carrier Sensing with Backoff:** Before transmitting, a station senses the medium. If busy, it waits. When the medium becomes idle, the station waits for a defined inter-frame spacing period (IFS), then enters a random backoff period (countdown timer). The station only transmits when the timer reaches zero and the medium is still idle.

2. **RTS/CTS Handshake (Request to Send / Clear to Send):**
   - The sender first transmits a short RTS frame to the receiver.
   - If the receiver is ready and the medium is clear, it responds with a CTS frame.
   - All other stations that hear either the RTS or CTS set their Network Allocation Vector (NAV) — a virtual carrier sensing mechanism — indicating the medium will be busy for a specified duration.
   - Only then does the sender transmit the data frame.
   - This solves the hidden terminal problem.

3. **ACK Mechanism:** The receiver sends an acknowledgment (ACK) after successfully receiving a data frame. If no ACK is received, the sender assumes a collision occurred and retransmits after a random backoff.

4. **NAV (Network Allocation Vector):** A timer maintained by each station that indicates how long the medium will be reserved. Stations defer transmission while their NAV is non-zero, even without physically sensing the medium as busy.

### (c) CPU Components and Instruction Execution [6 marks]

**CPU Diagram:**

```
┌─────────────────────────────────────────────────────────┐

│                                                         │
│  ┌──────┐     ┌──────┐     ┌──────┐     ┌──────────┐  │
│  │  PC  │────▶│  MAR │────▶│      │     │Execution │  │
│  │      │     │      │     │      │     │  Unit    │  │
│  └──────┘     └──────┘     │      │     │          │  │
│                    ▲        │Memory│     │  ┌─────┐ │  │
│  ┌──────┐         │        │ Bus  │     │  │ ALU │ │  │
│  │  IR  │◀────────┼────────│      │     │  └─────┘ │  │
│  │      │         │        │      │     │          │  │
│  └──────┘     ┌──────┐     │      │     │  ┌─────┐ │  │
│      │        │  MBR │◀───▶│      │     │  │Regs │ │  │
│      │        │      │     │      │     │  └─────┘ │  │
│      ▼        └──────┘     └──────┘     └──────────┘  │
│  [Decode]                                              │
│                                                         │
└─────────────────────────────────────────────────────────┘
                        │
                   System Bus
                        │
                   ┌─────────┐
                   │  Memory │
                   └─────────┘
```

**Component descriptions:**

- **PC (Program Counter):** Holds the address of the next instruction to be fetched from memory. It is automatically incremented after each fetch.

- **MAR (Memory Address Register):** Holds the memory address to be accessed (either for reading or writing). It connects to the address bus.

- **MBR (Memory Buffer Register) / MDR (Memory Data Register):** Holds the data being transferred to or from memory. It connects to the data bus. Data read from memory is placed here; data to be written to memory is placed here first.

- **IR (Instruction Register):** Holds the current instruction that has been fetched from memory and is being decoded/executed.

- **Execution Unit:** Contains the ALU (Arithmetic Logic Unit) and general-purpose registers. Performs the actual computation specified by the instruction.

**How instructions are executed (Fetch-Decode-Execute cycle):**

1. **Fetch:**
   - The content of the PC is copied into the MAR (MAR ← PC).
   - A memory read is initiated; the instruction at the address in MAR is transferred via the data bus into the MBR (MBR ← Memory[MAR]).
   - The content of the MBR is copied into the IR (IR ← MBR).
   - The PC is incremented to point to the next instruction (PC ← PC + 1).

2. **Decode:**
   - The control unit decodes the instruction in the IR.
   - It identifies the opcode (what operation to perform) and the operand(s)/addressing mode.

3. **Execute:**
   - If the operand needs to be fetched from memory, the operand address is placed in the MAR, and the data is fetched into the MBR.
   - The Execution Unit performs the operation (e.g., ALU performs arithmetic/logic).
   - Results are stored in the appropriate register or written back to memory (via MBR and MAR).

4. The cycle then repeats from step 1.

### (e) Binary Division: 12 / 2 [8 marks]

**Decimal:** 12 / 2 = 6

**Binary:** 12 = 01100, 2 = 00010

Using the restoring division algorithm with 5-bit registers:
- B (Dividend) = 01100
- Y (Divisor) = 00010
- A (Accumulator) = 00000 (initially zero)
- Count = 5 (number of bits in dividend)

**Algorithm steps:**
1. Shift A and B left together (A gets the MSB of B, B shifts left).
2. Subtract Y from A (A ← A - Y).
3. If A is negative (MSB = 1): restore A (A ← A + Y), set LSB of B to 0.
4. If A is non-negative: set LSB of B to 0... actually set quotient bit to 1 (LSB of B = 1).
5. Decrement count. Repeat until count = 0.

| | Restore: A = A + Y | 00000 | 11000 | | Set B[0] = 0 |
| | | 00000 | 11000 | 4 | |

### (f) Direct vs Indirect Addressing [2 marks]

**Direct Addressing:**
- The address field in the instruction contains the actual (effective) memory address of the operand.
- Only ONE memory reference is needed to fetch the operand (the instruction gives the address directly, and one memory access retrieves the data).
- Limitation: address space is limited by the size of the address field.

**Indirect Addressing:**
- The address field in the instruction contains the address of a memory location that holds the effective address of the operand.
- TWO memory references are needed: one to fetch the pointer (the address stored at the given location), and a second to fetch the actual operand from the effective address.
- Advantage: can access a larger address space (the pointer in memory can be wider than the address field in the instruction).

## Question 3: Interrupts, I/O and Instruction Pipelining (25 marks)

### (a) Two Drawbacks of Interrupt-Driven I/O [4 marks]

Although interrupt-driven I/O is more efficient than programmed I/O (because the processor can do other work while waiting), it still has two significant drawbacks:

**Drawback 1: Processor is involved in every data transfer**
- Every word/byte of data transferred between the I/O device and memory must pass through the processor. The processor must execute instructions to move data from the I/O module's data register to memory (or vice versa).
- This consumes processor cycles for every unit of data transferred. For high-speed devices or bulk transfers (e.g., disk I/O), this becomes a significant overhead, slowing down both the I/O transfer and the processor's ability to execute other programs.

**Drawback 2: Context switching overhead**
- Every interrupt requires the processor to save its current state (registers, program counter, status flags), process the interrupt (transfer data), and then restore the previous state to resume the interrupted program.
- This context switching consumes time and processor resources. If interrupts are frequent (e.g., one per byte on a fast device), the overhead of repeated context switches can degrade system performance significantly, potentially worse than programmed I/O.

*Note: DMA (Direct Memory Access) solves both drawbacks by allowing the I/O module to transfer data directly to/from memory without processor intervention for each word.*

### (b) Data Hazards [8 marks]

#### (i) What is a data hazard? Hardware technique to overcome it [4 marks]

**Data Hazard:**
A data hazard occurs in a pipelined processor when an instruction depends on the result of a previous instruction that has not yet completed its execution. The dependent instruction cannot proceed because the data it needs is not yet available. This creates a pipeline stall (bubble) that reduces the performance gained from pipelining.

There are three types:
- **RAW (Read After Write):** An instruction tries to read a source before a prior instruction has written to it.
- **WAR (Write After Read):** An instruction tries to write to a destination before a prior instruction has read from it.
- **WAW (Write After Write):** An instruction tries to write to a destination before a prior instruction has written to it.

**Hardware Technique: Data Forwarding (Bypassing / Short-circuiting)**

Data forwarding overcomes most data hazards by:
- Adding special hardware paths (forwarding paths/bypass paths) that route the result of an ALU operation directly from the output of the execution stage back to the input of the execution stage (or other needed stage).
- Instead of waiting for the result to be written back to the register file (WB stage) and then read in the next instruction's decode stage, the result is "forwarded" as soon as it is computed.
- This eliminates the need to stall in most RAW hazard situations.
- The forwarding unit detects when a source operand register of the current instruction matches the destination register of a preceding instruction still in the pipeline, and selects the forwarded value instead of the register file value.

#### (ii) Register Data Dependencies [4 marks]

Given instructions:

### (c) Multiple Device Interrupts: Priority Handling [8 marks]

When multiple device interrupts occur simultaneously or while another interrupt is being processed, the processor must determine which interrupt to service. There are several mechanisms:

**1. Software Polling:**
- When an interrupt occurs, the processor executes a polling routine that checks each device's status register in a fixed, predetermined order.
- The first device found requesting service is handled.
- Priority is determined by the position in the polling sequence (devices polled first have higher priority).
- Simple but slow — must check devices sequentially.

**2. Hardware Priority (Daisy-Chain):**
- Devices are connected in series (daisy-chain) to the interrupt acknowledge line.
- The interrupt acknowledge signal passes through each device in sequence.
- The device closest to the processor on the chain has highest priority.
- When a device receives the acknowledge and has a pending interrupt, it blocks the signal from propagating further and places its interrupt vector on the data bus.
- Fast but inflexible — priority is fixed by physical connection.

```
CPU ──ACK──▶ Device 1 ──ACK──▶ Device 2 ──ACK──▶ Device 3
  (highest priority)                        (lowest priority)
```

**3. Vectored Interrupts with Priority Levels:**
- Each device has a priority level assigned.
- When multiple interrupts are pending, hardware (interrupt controller) compares priorities and presents the highest-priority interrupt to the processor.
- The processor compares the incoming interrupt priority with its current execution priority (stored in the processor status register).
- If the new interrupt has higher priority, the current execution is suspended to service it (nested interrupts).
- Lower-priority interrupts are held pending until higher-priority processing completes.

**4. Programmable Interrupt Controller (PIC):**
- A dedicated chip (e.g., Intel 8259A) manages multiple interrupt inputs.
- It has a priority resolver that determines which pending interrupt to signal to the CPU.
- Priorities can be configured by software (fixed, rotating, or specific priority modes).
- Handles interrupt masking (specific interrupts can be disabled).
- Supports interrupt nesting.

```
Device 0 ──┐
Device 1 ──┤
Device 2 ──┼──▶ [PIC / Interrupt Controller] ──INT──▶ CPU
Device 3 ──┤         (priority resolver,
   ...     ┤          mask register)
Device n ──┘
```

**5. Interrupt Masking:**
- The processor status word contains an interrupt priority level.
- Only interrupts with priority higher than the current level are serviced.
- Lower-priority interrupts are masked (disabled) and remain pending.
- This enables nested/prioritised interrupt handling.

**Decision process when multiple interrupts are pending:**
1. Interrupt controller receives signals from multiple devices.
2. Priority resolver determines the highest-priority pending interrupt.
3. If its priority exceeds the processor's current interrupt level, it signals the processor.
4. The processor acknowledges, saves context, and begins the ISR for the highest-priority device.
5. Other pending interrupts wait or can nest if they have even higher priority.

### (d) What does the processor do when DMA controls the bus? [3 marks]

When the DMA (Direct Memory Access) module takes control of the system bus, the processor is temporarily unable to access the bus. This is known as **cycle stealing**. During this time:

1. **The processor is suspended from bus access but continues internal operations.** The processor can continue executing instructions that do not require the system bus — for example, operations involving only internal registers, or instructions already in the instruction cache/pipeline that don't need memory access.

2. **The processor enters a wait state for bus operations.** If the processor needs to access memory (for fetching instructions or data), it must wait until the DMA module releases the bus. The DMA module typically uses the bus for only one or a few bus cycles at a time (transferring one word), then returns control to the processor.

3. **The processor is NOT interrupted.** Unlike interrupt-driven I/O, no interrupt is generated and no context saving occurs. The processor simply pauses bus access momentarily. The DMA module "steals" individual bus cycles, which slightly slows the processor but is far more efficient than having the processor handle every data transfer.

When the entire DMA transfer is complete (the full block of data has been moved), the DMA module then generates an interrupt to notify the processor that the transfer is finished.

### (e) Multicore Processor vs Coprocessor [2 marks]

**Multicore Processor:**
- A single chip containing two or more independent processing cores (CPUs).
- Each core can execute its own instruction stream independently and simultaneously.
- All cores are general-purpose and can run any type of instruction.
- Cores share main memory and typically some level of cache.
- Used for parallel execution of multiple threads/processes (true parallelism).
- Examples: Intel Core i7 (4-8 cores), AMD Ryzen (6-16 cores).

**Coprocessor:**
- A supplementary processor designed to handle specific, specialised tasks to offload work from the main CPU.
- It cannot operate independently — it works alongside and under the direction of the main CPU.
- It has specialised hardware optimised for particular operations (e.g., floating-point arithmetic, graphics, encryption).
- The main CPU delegates specific instructions to the coprocessor when encountered.
- Examples: floating-point coprocessor (Intel 8087), GPU as a coprocessor, cryptographic coprocessor.

**Key difference:** A multicore processor has multiple independent general-purpose cores that can each run complete programs, while a coprocessor is a specialised, dependent processor that handles only specific operations delegated by the main CPU.
