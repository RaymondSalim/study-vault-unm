## Question 1: Networks (25 marks)

### (a) Differentiate between the following terms [8 marks]

#### (i) Thicknet and Thinnet

| Feature | Thicknet (10Base5) | Thinnet (10Base2) |
|---------|-------------------|-------------------|
| Cable diameter | ~10mm thick coaxial cable | ~5mm thin coaxial cable |
| Maximum segment length | 500 metres | 185 metres |
| Maximum nodes per segment | 100 | 30 |
| Data rate | 10 Mbps | 10 Mbps |
| Connector type | Vampire tap / AUI connector | BNC T-connector |
| Flexibility | Rigid and difficult to install | More flexible and easier to handle |
| Cost | More expensive | Less expensive |

**Thicknet** uses a thick coaxial cable (RG-8) primarily used as a network backbone. It supports longer distances but is harder to work with due to its rigidity.

**Thinnet** uses a thinner coaxial cable (RG-58) that is cheaper and easier to install. It connects directly to the network interface card via a BNC connector but supports shorter distances and fewer nodes.

#### (ii) Simplex and Full Duplex Communications

**Simplex communication** is unidirectional -- data can only flow in one direction. The sender can only send and the receiver can only receive. Example: television broadcasting, where signals travel from the station to the receiver only.

**Full duplex communication** is bidirectional and simultaneous -- data can flow in both directions at the same time. Both parties can send and receive concurrently. Example: a telephone conversation, where both parties can speak and listen simultaneously.

| Feature | Simplex | Full Duplex |
|---------|---------|-------------|
| Direction | One-way only | Both directions simultaneously |
| Channel usage | One channel | Two channels (or shared) |
| Example | TV broadcast, keyboard input | Telephone, modern Ethernet |
| Bandwidth utilisation | Less efficient | More efficient |

#### (iii) Broadband and Baseband Coaxial Cables

**Baseband** transmission uses the entire bandwidth of the cable for a single digital signal. It employs digital signalling where the signal occupies the full frequency spectrum of the medium. Only one signal can be transmitted at a time (time-division multiplexing may be used). Example: Ethernet LANs.

**Broadband** transmission divides the cable bandwidth into multiple frequency bands, allowing several analogue signals to be transmitted simultaneously using frequency-division multiplexing (FDM). Each channel operates at a different frequency. Example: cable television systems.

| Feature | Baseband | Broadband |
|---------|----------|-----------|
| Signal type | Digital | Analogue |
| Number of channels | Single channel | Multiple channels |
| Multiplexing | TDM | FDM |
| Direction | Bidirectional | Unidirectional (requires two cables or split bandwidth) |
| Distance | Limited (up to a few km) | Greater distances (up to tens of km) |
| Cost | Less expensive | More expensive |

#### (iv) Bit Rate and Baud Rate

**Bit rate** is the number of bits transmitted per second (bps). It measures the actual data throughput of a communication channel.

**Baud rate** is the number of signal changes (symbols) per second. It measures the number of times the signal state changes on the transmission medium per second.

The relationship between them is:

```
Bit rate = Baud rate x Number of bits per signal element (symbol)
```

If each signal element encodes only 1 bit, then bit rate = baud rate. However, with advanced modulation techniques (e.g., QAM), each signal change can represent multiple bits, making bit rate > baud rate.

**Example:** If a modem uses 4 different voltage levels (2 bits per symbol) and operates at 1000 baud, the bit rate is 2000 bps.

### (b) Packet Switching Network using Datagram Approach [6 marks]

#### Operation of Datagram Packet Switching

In a datagram approach to packet switching, each packet (datagram) is treated as an independent unit that contains full addressing information. The operation proceeds as follows:

1. **Message division:** The source breaks the message into smaller packets, each with a header containing the source address, destination address, sequence number, and other control information.

2. **Independent routing:** Each packet is routed independently through the network. Different packets from the same message may take different paths to reach the destination.

3. **Store-and-forward:** At each intermediate node (router), the packet is received in full, stored temporarily, the routing table is consulted, and the packet is forwarded to the next appropriate node.

4. **Reassembly:** At the destination, packets may arrive out of order. The destination node uses sequence numbers to reassemble the packets into the original message.

```
Diagram: Datagram Packet Switching

Source                                              Destination
  |                                                      |
  |    +--------+      +--------+      +--------+       |
  |--->| Node A |----->| Node B |----->| Node D |------>|  (Packet 1: path A-B-D)
  |    +--------+      +--------+      +--------+       |
  |         |                               ^            |
  |         v                               |            |
  |    +--------+      +--------+           |            |
  |--->| Node A |----->| Node C |---------->|----------->|  (Packet 2: path A-C-D)
  |    +--------+      +--------+                        |
  |                                                      |

  Packets 1, 2, 3... each independently routed.
  May arrive out of order at destination.
```

#### Advantage: Flexibility and Robustness

If a node or link fails during transmission, subsequent packets can be dynamically routed through alternative paths. There is no need to re-establish a path, making the network resilient to partial failures. Resources are also used more efficiently because bandwidth is shared among all users on demand.

#### Disadvantage: Overhead and Out-of-Order Delivery

Each packet must carry full destination addressing information, increasing overhead. Packets may arrive out of order or be lost, requiring the destination to perform reordering and request retransmission. This adds complexity to the receiving end and may introduce variable delay (jitter), making it less suitable for real-time applications.

### (c) Adaptive vs Non-Adaptive Routing Algorithms [5 marks]

#### Non-Adaptive Routing (Static Routing)

Non-adaptive routing algorithms do not base their routing decisions on measurements or estimates of current network traffic or topology. The routes are computed in advance (offline) and loaded into the routers before the network starts operating. The routing table remains fixed regardless of changes in network conditions.

**Examples:** Flooding, shortest path routing (pre-computed).

**Characteristics:**
- Routing decisions are fixed and do not change with network conditions
- Simple to implement
- Low computational overhead on routers
- Cannot respond to congestion or link failures

#### Adaptive Routing (Dynamic Routing)

Adaptive routing algorithms modify their routing decisions based on changes in network topology and traffic conditions. They gather information about the current state of the network (e.g., link delays, queue lengths, failures) and adjust routes accordingly.

**Examples:** Distance vector routing (RIP), link-state routing (OSPF).

**Characteristics:**
- Routing decisions change dynamically based on network state
- More complex to implement
- Higher computational and communication overhead
- Can optimise performance and respond to failures

#### Why Adaptive Routing is Superior

Adaptive routing is superior because:

1. **Fault tolerance:** It can detect link or node failures and reroute traffic around them automatically, maintaining network connectivity.
2. **Load balancing:** It can distribute traffic across multiple paths to avoid congestion, improving overall throughput.
3. **Performance optimisation:** It responds to changing traffic patterns, reducing delays by avoiding congested areas.
4. **Scalability:** It handles network growth and changes without manual reconfiguration.

Non-adaptive routing cannot respond to failures or congestion, leading to packet loss and poor performance when network conditions change.

### (d) Define the term "Flooding" [2 marks]

**Flooding** is a non-adaptive routing technique in which every incoming packet is sent out on every outgoing link except the one on which it arrived. This ensures that the packet will eventually reach its destination (if a path exists) because it explores all possible routes simultaneously.

To prevent infinite circulation of packets, control mechanisms are used:
- **Hop counter:** Each packet carries a counter that is decremented at each hop; the packet is discarded when the counter reaches zero.
- **Sequence numbers:** Each source-destination pair maintains sequence numbers so that duplicate packets can be identified and discarded.

Flooding is impractical for general data transfer due to the enormous number of duplicate packets generated, but it is useful for broadcasting routing information and as a benchmark for comparing other routing algorithms (it always finds the shortest path).

### (e) Operations of the Two Major Types of Bridges [4 marks]

#### 1. Transparent Bridge (Learning Bridge)

Transparent bridges operate in a manner that is invisible to the connected network stations -- no configuration of hosts is needed.

**Operation:**
- The bridge maintains a **forwarding table** (MAC address table) that maps MAC addresses to ports.
- When a frame arrives, the bridge examines the source address and records which port it came from (learning).
- The bridge then checks the destination address:
  - If the destination is on the same port as the source, the frame is **filtered** (not forwarded).
  - If the destination is known to be on a different port, the frame is **forwarded** only to that port.
  - If the destination is unknown, the frame is **flooded** to all ports except the source port.
- Over time, the bridge builds a complete picture of which addresses are reachable through which ports.

#### 2. Source Routing Bridge

Source routing bridges are primarily used in Token Ring networks. The routing decision is made by the source station, not the bridge.

**Operation:**
- The source station determines the complete route (sequence of bridges and LANs) before sending data.
- To discover the route, the source sends a **discovery frame** (explorer frame) that is broadcast across all bridges and LANs.
- Each bridge that forwards the discovery frame appends its bridge ID and the LAN ID to the frame's routing information field.
- When the discovery frame reaches the destination, the destination sends a response back along the recorded path.
- The source then includes this complete route in the header of all subsequent data frames.
- Bridges examine the routing information in each frame to determine whether to forward it and to which port.

## Question 2: Binary Arithmetic, Processor Registers and Addressing Modes (25 marks)

### (a) Shift Operations on 8-bit Register [4 marks]

**Initial value:** `01111011`

We perform the following operations in sequence:

#### (i) SHR (Logical Shift Right)

Shift all bits one position to the right. The leftmost bit is filled with 0, and the rightmost bit is shifted out (into the carry flag).

```
Before: 0 1 1 1 1 0 1 1
After:  0 0 1 1 1 1 0 1    Carry = 1
```

**Result: `00111101`**, Carry = 1

#### (ii) CIR (Circular/Rotate Right)

Rotate all bits one position to the right. The rightmost bit wraps around to the leftmost position (through carry in some architectures). Assuming CIR rotates through carry:

```
Before: 0 0 1 1 1 1 0 1    Carry(in) = 1
After:  1 0 0 1 1 1 1 0    Carry(out) = 1
```

**Result: `10011110`**, Carry = 1

*Note: If CIR is a simple rotate (not through carry), the rightmost bit goes directly to the leftmost position:*

```
Before: 0 0 1 1 1 1 0 1
After:  1 0 0 1 1 1 1 0    (bit 0 wraps to bit 7)
```

**Result: `10011110`** (same result in this case since carry was 1)

#### (iii) A SHR (Arithmetic Shift Right)

Shift all bits one position to the right, but preserve the sign bit (MSB). The MSB is replicated, and the rightmost bit shifts out.

```
Before: 1 0 0 1 1 1 1 0
After:  1 1 0 0 1 1 1 1    Carry = 0
```

**Result: `11001111`**, Carry = 0

#### (iv) SHR (Logical Shift Right)

Shift all bits one position to the right. The leftmost bit is filled with 0.

```
Before: 1 1 0 0 1 1 1 1
After:  0 1 1 0 0 1 1 1    Carry = 1
```

**Result: `01100111`**, Carry = 1

#### Summary of Sequential Operations:

| Step | Operation | Result |
|------|-----------|--------|
| Start | — | `01111011` |
| (i) | SHR | `00111101` |
| (ii) | CIR | `10011110` |
| (iii) | A SHR | `11001111` |
| (iv) | SHR | `01100111` |

### (b) IEEE 754 Representation of 12.125 [8 marks]

#### Converting 12.125 to Binary

**Integer part (12):**
- 12 / 2 = 6 remainder 0
- 6 / 2 = 3 remainder 0
- 3 / 2 = 1 remainder 1
- 1 / 2 = 0 remainder 1

Integer part = `1100`

**Fractional part (0.125):**
- 0.125 x 2 = 0.25 --> 0
- 0.25 x 2 = 0.5 --> 0
- 0.5 x 2 = 1.0 --> 1

Fractional part = `001`

**Combined:** 12.125 in binary = `1100.001`

**Normalised:** `1.100001 x 2^3`

| Field | Bits | Value |
|-------|------|-------|
| Sign | 1 bit | 0 (positive) |

### (c) Function of CPU Registers [10 marks]

#### (i) PC - Program Counter (2 marks)

The **Program Counter** (also called the Instruction Pointer) holds the memory address of the next instruction to be fetched from memory. After each instruction is fetched, the PC is automatically incremented to point to the next sequential instruction. When a branch or jump instruction is executed, the PC is loaded with the target address rather than simply being incremented.

**Function:** Keeps track of program execution sequence, ensuring instructions are fetched in the correct order. It is essential for the fetch phase of the instruction cycle.

#### (ii) IR - Instruction Register (2 marks)

The **Instruction Register** holds the current instruction that has been fetched from memory and is being decoded/executed. Once an instruction is fetched from the memory location pointed to by the PC, it is placed in the IR where the control unit can decode the opcode and operand fields.

**Function:** Stores the current instruction during the decode and execute phases so the control unit can interpret the opcode and determine what operation to perform and on what operands.

#### (iii) MAR - Memory Address Register (2 marks)

The **Memory Address Register** holds the address of the memory location that is about to be accessed (either for a read or write operation). It is connected to the address bus. When the CPU needs to fetch an instruction or data, the relevant address is placed in the MAR before the memory access occurs.

**Function:** Provides the address to memory for read/write operations. It acts as the interface between the CPU and the address bus, specifying which memory location to access.

#### (iv) MBR - Memory Buffer Register (2 marks)

The **Memory Buffer Register** (also called the Memory Data Register, MDR) holds the data that has just been read from memory or the data that is about to be written to memory. It is connected to the data bus and acts as a buffer between the CPU and main memory.

**Function:** Serves as an intermediary for all data transfers between the CPU and memory. Data read from memory is placed here before being used by the CPU, and data to be written is placed here before being sent to memory.

#### (v) ACC - Accumulator (2 marks)

The **Accumulator** is a general-purpose register that stores intermediate results of arithmetic and logic operations performed by the ALU. In simple CPU architectures, it serves as one of the implicit operands and the destination for ALU results. For example, an ADD instruction might add the contents of a memory location to the accumulator and store the result back in the accumulator.

**Function:** Holds operands and results of ALU operations. It is the primary working register in accumulator-based architectures and participates directly in most arithmetic and logic computations.

### (d) Direct vs Indirect Addressing [3 marks]

#### Direct Addressing

In **direct addressing**, the address field in the instruction contains the actual (effective) memory address of the operand. The CPU goes directly to the specified address to fetch the data.

```
Instruction: [Opcode | Address A]
Operand is at memory location A
```

**Memory references required: 1** -- one access to fetch the operand from the address specified in the instruction (in addition to the instruction fetch itself).

#### Indirect Addressing

In **indirect addressing**, the address field in the instruction contains the address of a memory location that itself holds the actual address of the operand. The CPU must first access the pointer address to obtain the effective address, then access the effective address to get the operand.

```
Instruction: [Opcode | Address A]
Memory[A] contains address B
Operand is at memory location B
```

**Memory references required: 2** -- one access to fetch the pointer from address A, and a second access to fetch the actual operand from address B.

#### Key Differences

| Feature | Direct | Indirect |
|---------|--------|----------|
| Address field contains | Effective address of operand | Address of pointer to operand |
| Memory accesses for operand | 1 | 2 |
| Address space | Limited by address field size | Can access larger address space |
| Flexibility | Fixed address | Pointer can be modified at runtime |
| Speed | Faster (fewer memory accesses) | Slower (extra memory access) |

## Question 3: Interrupts, I/O and Instruction Pipelining (25 marks)

### (a) Instruction Pipelining [2 marks]

**Instruction pipelining** is a technique used in processor design to increase instruction throughput by overlapping the execution of multiple instructions. The instruction cycle is divided into a series of stages (such as Fetch, Decode, Execute, Memory Access, Write-back), and multiple instructions are processed simultaneously, each at a different stage.

Like an assembly line in a factory, while one instruction is being executed, the next instruction is being decoded, and a third is being fetched. This allows the processor to complete one instruction per clock cycle (in ideal conditions) even though each individual instruction takes multiple cycles to complete. The key principle is that different hardware resources are utilised at each stage, enabling parallel processing of multiple instructions.

---

### (b) Data Dependency [2 marks]

A **data dependency** occurs when an instruction in a pipeline depends on the result of a previous instruction that has not yet completed its execution. This means the dependent instruction cannot proceed until the instruction it depends on produces its result.

Data dependencies create **pipeline hazards** that can stall the pipeline, reducing the performance gains that pipelining provides. The processor must either wait (insert bubble/stall cycles), forward the result early (data forwarding/bypassing), or reorder instructions to avoid the hazard.

---

### (c) Three Classes of Data Dependencies [6 marks]

#### 1. RAW (Read After Write) - True Dependency

A **RAW dependency** occurs when an instruction needs to read a value that a previous instruction has not yet written. The second instruction must wait until the first instruction writes its result.

**Example:**
```assembly
I1: ADD R1, R2, R3    ; R1 = R2 + R3  (writes to R1)
I2: SUB R4, R1, R5    ; R4 = R1 - R5  (reads R1 -- depends on I1's result)
```

Instruction I2 cannot read R1 until I1 has completed writing to R1. If I2 reaches the execute stage before I1 has written its result to R1, I2 will read a stale value.

#### 2. WAR (Write After Read) - Anti-Dependency

A **WAR dependency** occurs when an instruction needs to write to a location that a previous instruction has not yet finished reading. The second instruction must not overwrite the value before the first instruction has read it.

**Example:**
```assembly
I1: ADD R1, R2, R3    ; R1 = R2 + R3  (reads R3)
I2: SUB R3, R4, R5    ; R3 = R4 - R5  (writes to R3 -- must not overwrite before I1 reads)
```

If I2 writes to R3 before I1 has read R3, I1 will get the wrong value. This is called an anti-dependency because it is the reverse of the natural flow of data.

#### 3. WAW (Write After Write) - Output Dependency

A **WAW dependency** occurs when two instructions both write to the same location. They must write in the correct program order to ensure the final value is correct.

**Example:**
```assembly
I1: ADD R1, R2, R3    ; R1 = R2 + R3  (writes to R1)
I2: SUB R1, R4, R5    ; R1 = R4 - R5  (also writes to R1)
```

If I2 completes before I1 (possible in out-of-order execution), then R1 would contain the result of I1 instead of I2, which is incorrect. The final value of R1 must be the result of I2 since it comes later in program order.

### (d) Handling Multiple Device Interrupts [8 marks]

When multiple devices generate interrupts simultaneously, the processor must determine which one to service first. There are several methods:

#### 1. Priority-Based Interrupt Handling

Each device is assigned a **priority level**. When multiple interrupts occur simultaneously, the processor services the highest-priority interrupt first. This can be implemented through:

**Software Polling:**
- When an interrupt occurs, the processor executes a polling routine that checks each device's status register in a predetermined order (priority order).
- The first device found with a pending interrupt is serviced.
- Simple to implement but slow because every device must be checked sequentially.
- The priority is determined by the position in the polling sequence.

**Hardware Priority (Daisy Chain):**
- Devices are connected in a daisy-chain configuration, where the interrupt acknowledge signal passes through devices in priority order.
- The device closest to the processor in the chain has the highest priority.
- When an interrupt occurs, the acknowledge signal propagates down the chain; the first device with a pending interrupt captures the acknowledge and identifies itself.
- Fast but inflexible -- priority is fixed by physical position.

**Vectored Priority Interrupt:**
- Each device provides a unique interrupt vector (address) that directs the processor to the appropriate interrupt service routine (ISR).
- A hardware priority encoder determines which of the simultaneous requests has the highest priority.
- The vector of the highest-priority device is sent to the processor.
- Fast and flexible.

#### 2. Interrupt Masking

- The processor can selectively **mask** (disable) lower-priority interrupts while servicing a higher-priority one.
- A priority mask register determines which interrupt levels are currently enabled.
- Higher-priority interrupts can preempt (interrupt) lower-priority ISRs, creating nested interrupts.

#### 3. Priority Interrupt Controller (e.g., PIC 8259)

- A dedicated hardware controller manages multiple interrupt lines.
- It resolves priority, masks lower-priority interrupts, and presents the highest-priority interrupt vector to the processor.
- Can be programmed for fixed priority, rotating priority, or other schemes.

**Overall process when multiple interrupts occur:**
1. Multiple devices assert their interrupt lines.
2. The priority mechanism (hardware or software) identifies the highest-priority request.
3. Lower-priority interrupts are held pending (masked or queued).
4. The processor saves its current state and jumps to the ISR for the highest-priority device.
5. After completing the ISR, the processor checks for any remaining pending interrupts and services the next highest-priority one.

### (e) Processor Activity During DMA Bus Control [3 marks]

When the **DMA (Direct Memory Access) module** takes control of the system bus, the processor is temporarily disconnected from the bus -- a condition known as **cycle stealing**. During this time:

1. **The processor cannot access main memory** because it does not have control of the address and data buses. Any memory read/write operations by the CPU are suspended.

2. **The processor continues internal operations** that do not require bus access. It can:
   - Continue executing instructions already in the pipeline that do not need memory access
   - Perform ALU operations on data already in registers
   - Decode instructions already fetched into the instruction register
   - Process micro-operations that are internal to the CPU

3. **The processor is not idle** -- it productively uses time on internal computations while the DMA controller handles the data transfer between I/O devices and memory without CPU intervention.

This arrangement is advantageous because it allows high-speed block data transfers (e.g., disk to memory) without burdening the processor with handling each individual word transfer, significantly improving overall system throughput.

### (f) Four Characteristics of RISC Architectures [4 marks]

#### 1. Simple, Fixed-Length Instructions

All instructions are the same length (typically 32 bits), which simplifies instruction fetch and decode logic. This enables faster decoding and facilitates pipelining since instruction boundaries are predictable and uniform.

#### 2. Load/Store Architecture

Only dedicated LOAD and STORE instructions access memory. All other operations (arithmetic, logic, comparison) operate exclusively on data in registers. This separation simplifies the instruction set and the hardware, and makes pipelining more efficient since memory access is isolated to specific stages.

#### 3. Large Register File

RISC processors have a large number of general-purpose registers (typically 32 or more). This reduces the need for frequent memory accesses, as more variables and intermediate results can be held in fast registers. The large register file compensates for the load/store restriction by keeping frequently-used data readily available.

#### 4. Single-Cycle Execution

Most instructions complete in a single clock cycle (or a single pipeline stage per clock). This uniformity allows for efficient pipelining with minimal stalls. Complex operations that would require multiple cycles in CISC are broken down into sequences of simple instructions, each completing in one cycle.

**Additional characteristics (for completeness):**
- Hardwired control unit (no microcode)
- Heavy reliance on compiler optimisation
- Few addressing modes
- Fixed instruction format with regular field placement
- Emphasis on instruction pipelining efficiency
